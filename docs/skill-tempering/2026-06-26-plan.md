# skill-tempering 实施计划（本轮：底座 + 短路径 + 骨架 · 精简 6 文件）

> **For agentic workers:** REQUIRED SUB-SKILL: 用 superpowers:subagent-driven-development（推荐）或 superpowers:executing-plans 逐任务实施。步骤用 `- [ ]` 勾选跟踪。
>
> **这是 skill 创作计划（产物是 markdown skill 文件，非代码）**——TDD 仪式改写为「按设计稿权威源撰写 → 对验收判据核 → commit」；"测试"＝可核验的验收判据，capstone＝拿 skill-tempering 审它自己（§6 自我 dogfood）。

**Goal:** 在 `incubating/skill-tempering/` 建出一个自包含 meta-skill，能用「短路径·审计透镜」静态体检任意已有 skill。

**Architecture:** 脊柱（SKILL.md）只路由 + 细则下沉 references；底座 = 7 条机制层通用核 + 一道现场智能派生工序；本轮只实现短路径，长路径不建文件（foothold 在设计稿 §3.2）。

**Tech Stack:** 纯 markdown skill（无运行时代码）；遵 Claude Agent Skills 结构（frontmatter + SKILL.md + references/ 渐进披露）。

**权威内容源（单一权威）：** 已 commit 的设计稿 [docs/skill-tempering/2026-06-26-design.md](./2026-06-26-design.md)。**本计划不复述设计稿正文**——各任务点名"依据 §X"取内容，计划只补 skill 化的撰写规则 + 验收。

## Global Constraints

逐条 verbatim，每个任务都隐含遵守：
- **全程中文**（思维、注释、文档）——`.claude/rules/principles.md`。
- **自包含·零运行期外部依赖**（#7 宪法级·不可 override）：零运行期外部 skill / 脚本 / 跨会话依赖。
- **脊柱只路由不复述**：SKILL.md 持跨路径脊柱（流程图 / ⏸ / 7 核名+一句 / 指针）；细则下沉 references；**指针只点该 reference 内非默认的具体动作**。
- **单一权威源**：每条判据 / 清单 / 动作只有一个家；禁两处都写。
- **守薄**：抗 sprawl / sediment / no-op / duplication。**出生从简（6 文件）**；README / CHANGELOG / BACKLOG 等随真实需要再长，**不在 v0 cargo-cult 成熟 skill 的终态结构**（早产的约定＝违 #5）。
- **通用核机制层措辞**：7 核不得带领域专属词（grep / 模块 / 幻灯片 / 代码…）。
- **不自动触发、显式调用**：description 写负向边界（只优化已有 skill；创建全新 skill 不在此列）。
- **长路径本轮不实现**：**不建桩文件**；SKILL.md 流程图标"长路径后续"，foothold 单一权威在设计稿 §3.2，MAINTAINING「搁置项」以指针指向之。

## File Structure（本轮要建的 6 个）

```
incubating/skill-tempering/
├── SKILL.md                    脊柱: 两路径流程图(长路径标"后续") + 7 核(名+一句+指针) + 派生工序概览 + ⏸ + 指针 + description
├── MAINTAINING.md              维护宪法(防膨胀/单一权威/结晶闸/毕业焙烧) + 「搁置项」(长路径 foothold 指针 + 4 开放问题)·不随运行载入
├── theory-foundation.md        理论骨(Huang et al./Böckeler/Anthropic + 6 先例借用)·研究支撑、不随运行载入
└── references/
    ├── principles.md           7 条通用核全文
    ├── derivation-protocol.md  派生工序(嗅探/失败形态/条件纪律种子库/critic/结晶治理/盲区声明)
    └── audit-lens.md           短路径细则 + 改进清单/部分符合声明 模板
```

每文件一个职责；`references/` 各文件是某路径才用的细则（渐进披露第三级）。

## §1 定位

**`skill-tempering`（淬炼）** 是一个 **meta-skill**：拿一份现成 skill，把它**优化 / 成熟**到本仓标准。

- 职责边界：只**优化 / 成熟已有 skill**（回火淬炼——实测到位、去脆性、变韧）；**不负责创建全新 skill**，那是另一回事、不在本 skill 范围。
- 层级：纯**维护层**机制——它是"作用于"一个 skill 的过程，**不是** skill 运行时"依赖"的库。它的产物是"一个更好的、仍然自包含的 skill"，故 structurally 不破被测 skill 的自包含。
- 触发：**不自动触发、由用户显式调用**（同 recon-driven-dev）。它是重型 meta 动作，误触发代价高。
- 发布：仓内独立 skill，走孵化炉模型（`incubating/` → 成熟 → filter-repo 毕业）。

### 两条路径（共用同一底座，短 ⊂ 长）

| | 短路径 · 审计透镜 | 长路径 · dogfood 成熟 |
|---|---|---|
| 输入 | 一份现成 skill（静态） | 一个 skill + 真实任务 |
| 动作 | 归型 → 跑通用核 + 派生纪律 → 完整性 critic → 产改进清单 | 实测 → 三分诊 → 处置 → 验证 → 沉淀 |
| 产出 | 改进清单（结构化部分符合声明） | 改进清单 + 回填覆盖账本 + 沉 CHANGELOG |
| 成本 | 便宜、单会话 | 贵、跨会话 |
| 关系 | = 长路径的"验证/审计"步独立跑 | 含短路径 |

## §2 底座：通用核 + 派生工序

底座**不是**一张平铺的原则清单（那会对不同型 skill 牛头不对马嘴），而是**「小而硬的通用核 + 一道现场智能派生工序」**——刻意**不做成机械分类器**。

> 为什么不是清单/分类器：枚举「skill 类型」或「判别轴」都被证明是**开放集**——每来一个新型 skill 就冒出新维度（见 §7.3/§7.4）。可机械兜住任意 skill 的固定脚手架不存在。这与 codebase-exploration 治"无穷栈"的结论同构：携带**通用槽 + 派生协议 + 种子（非穷举）**，而非硬编码全集。

### §2.1 通用核（7 条 · 机制层 · 任何 skill 都先过）

每条按机制层措辞（type-agnostic by construction），结构 = `原则 + 触发条件 + 理论骨 + 审计问句 + 强制力档`。**强制力档**（借 Sentinel）：`advisory / soft / hard`，默认见下，可现场覆盖（覆盖留痕）。

> 两个"硬"别混：**强制力档 hard** = 本次审计这条违规必改（针对被测 skill）；**「唯一硬约束」** = 自包含(#7)是连 meta-skill 作者改自己时都不可 override 的宪法级约束（针对 meta 层设计）。前者审计纪律、后者设计宪法，正交。

1. **自评不可信（generation ≠ judgment）** · `[universal · hard]`
   - 原则：模型对自己输出的正确性无可靠内部信号；产出与裁判必须分开。**承诺的形态决定验证的形态**——点真值→外部核那个点；完整性承诺→核全集；**无外部真值→诚实标低置信/待确认、交人，绝不自评放行、不假装完成**。
   - 理论骨：Huang et al.《LLMs Cannot Self-Correct Reasoning Yet》ICLR 2024（同行评审；内省自纠无外部信号常变差）。
   - 审计问句：这个 skill 在哪一步信了模型的自我断言？该步有外部真值吗——有就外部闸、无就退到诚实标注。
   - 收编：原 maker≠checker(A1) + 认识论诚实(A2) + 完整性闸(C7)。

2. **完成判据（completion criterion）** · `[universal · soft]`
   - 原则：显式、可判、带上限的"做到这儿就停"。含自检/迭代循环的须给死上限；无界循环或无判据是缺陷（够用即止 / 反 loopmaxxing）。
   - 理论骨：Anthropic「max iterations to maintain control」；anti-loopmaxxing（bdtechtalks）。
   - 审计问句：它界定了"到什么程度算完"吗？循环有上限吗？

3. **快乐路径之外（failure / absence path）** · `[universal · soft]`
   - 原则：为可预见的失败 / 前置缺失 / 能力不具备定义显式行为（降级或优雅退出），不默认一切顺利。
   - 审计问句：外部条件不具备时（工具缺失 / 输入异常 / 自检不收敛），它崩在快乐路径上还是优雅降级？

4. **确定性归工具、判断归模型** · `[universal · soft]`
   - 原则：能机械确定的别让模型猜（交确定性工具），需判断的别硬编码（留模型）。双向。
   - 理论骨：Böckeler computational vs inferential regulation。
   - 审计问句：有没有让模型做本可机械确定的活？或把需判断的硬编码成了死表？

5. **单一权威源 · 守薄** · `[universal · soft]`
   - 原则：每条判据 / 清单 / 动作只有一个权威家、禁复述；持续抗 sprawl / sediment / no-op / duplication 四类熵增。
   - 审计问句：同一判据有没有两处都写？有没有过时残留 / 空转问句？

6. **描述即激活闸** · `[universal · hard]`
   - 原则：name + description 决定 skill 能否在对的时机被唤起；写成"何时用 X"触发器（正向触发词 + 负向边界 / 反例分流），非"本 skill 做 X"摘要。
   - 理论骨：Anthropic skill 创作 best-practices（description 是 trigger 不是 summary；under-trigger 倾向）。
   - 审计问句：这个 description 能让它在对的时机被唤起吗？有负向边界吗？

7. **自包含 · 可移植** · `[universal · hard · 唯一硬约束]`
   - 原则：零运行期外部 skill / 脚本 / 跨会话依赖；改一个 skill 必从它自身长出。
   - 理论骨：可移植 / 可分发地基（npx skills 跨平台分发模型）。
   - 审计问句：运行期有没有依赖别的 skill / 外部脚本？改它时有没有借外部方法论？

### §2.2 派生工序（智能、非查表）

被测 skill 的"额外适用纪律"，**不靠枚举类型查表**，而是一道现场判断工序：

1. **归型 = 嗅 + 失败形态**（不问"你是哪类"）：
   - **从可观测证据自动嗅探**（借 Biome `domains`）：它产持久产物吗？产全称/穷尽断言吗？落不可逆副作用吗？多路径/多文件吗？价值在产物还是过程？——读这些**事实**，不贴类型标签。
   - **分叉轴 = 失败形态而非 skill 类型**（借 superpowers「Match the Form to the Failure」）：跑 baseline 看 agent 缺这个 skill 时怎么失败（skips-rule / wrong-shape / omits-element / condition-dependent——**小而封闭的集**），按失败形态选指导形式。

2. **派生条件纪律**（从 7 核 + 嗅到的证据推出，各带强制力档）：
   - 产全称断言 → 完整性验证形态（核全集）｜默认 soft
   - 高代价/不可逆决策 → HITL 人控闸（人放行）｜默认 hard
   - 持久产物 → 慢变·altitude + 可重生·新鲜度 ｜默认 soft
   - 多路径/多文件 → 分层披露（脊柱/references/指针）｜默认 soft
   - 副作用侧（凭证敏感 / blast-radius / 幂等重入）、输入侧（源受控性）、判据侧（时变/效度）…… **种子库非穷举**，缺了当场派生。
   - **边缘适用的打 advisory，不硬套**——这是软杀"牛头不对马嘴"的刀（边缘原则=建议，不是强制）。

3. **完整性 critic（我们独有、linter 没有的一环）**：显式问「这个 skill 有什么是我的种子没问到的命脉？」——种子不够就**当场派生新维度**。

4. **实证验真**（借 skill-creator / Anthropic）：派生出的判断别只靠 critic 内省，配一次 **baseline 双跑**（with / without skill）把"这条纪律对这个 skill 灵不灵"测出来。

5. **结晶（带治理闸）**：当场派生的新维度沉回种子库前——**默认 `pending` + 留痕 + 复核**（借 RuboCop `NewCops: pending` + Google arbiter）。**不自动回灌**（否则种子库无人负责地膨胀，违背"守薄"）。这是我们方案唯一没有现成先例、最该加治理的部件。

6. **盲区声明**：永不宣称"已覆盖所有 skill 类型"；产出附**结构化部分符合声明**（借 WCAG）：列〔未覆盖项 / 理由 / 责任人〕。

> §2 自洽性：底座对"skill 类型"这件事，正是用自己的①自评不可信（不合就标、当场派生不硬套）、④确定性归工具判断归模型（核机械、派生智能）、⑤守薄（声明盲区不假装穷尽）在运作——**它吃自己的狗粮**（见 §6）。

## §3 两路径细则

> **本轮范围**：实现 **底座（§2）+ 短路径（§3.1）+ 骨架（§4–6）**。**长路径（§3.2）本轮不做、后续单独讨论**——§3.2 只留已达成要点作 foothold，不进本轮实施。

### §3.1 短路径 · 审计透镜（`references/audit-lens.md`）
拿一份现成 skill，静态体检：归型（§2.2.1）→ 跑 7 通用核的审计问句 + 派生的条件纪律（各带档）→ 完整性 critic → 产**改进清单**（按强制力档分级：hard 违规必改、soft 默认改可 override、advisory 建议）+ 结构化盲区声明。单会话、便宜。

### §3.2 长路径 · dogfood 成熟（`references/dogfood-protocol.md`）

> **本轮不实现 · 后续单独讨论。** 下面是已啃下来的设计要点（foothold——供后续深化、别从零起）。

把 recon-driven-dev 的「实测打磨协议」通用化——**两会话刻意分开 + 覆盖账本驱动"何时算完"**：

- **阶段 0 起手**：给被测 skill 织一本**覆盖账本**（行为情境网格，织法见下）；可先跑一次短路径静态审计当起点。
- **阶段 1 实测会话**（真实项目里开 · **只跑不改**）：真实任务跑被测 skill + **baseline 双跑**(with/without 对照) + **exception-only 监督**(只记卡住 / 指令含糊 / 降级失败) + 末尾派**隔离复盘 sub-agent** 补抓缺陷 → 产改进点清单。**铁律：这会话绝不改 skill**（防自我合理化）。
- **阶段 2 打磨会话**（仓里另开 · 改 skill）：**三分诊**(skill 缺陷 / 任务特例 / 模型失误，只前者动 skill) → **处置门槛**(当场改 / 进 BACKLOG) → **改后验证 rubric**(＝拿短路径审计透镜核自己改动) → 回填覆盖账本 + 沉 CHANGELOG；**generalize-don't-overfit**。
- **收敛**：地板＝每格碰一次；天花板＝被碰的格连续 2 轮不冒新缺陷 ＝ 实测到位（自身也守 #2 完成判据）。

**两条承重设计**：① 两会话分开 + 隔离复盘 ＝ #1 自评不可信在打磨层（跑者 ≠ 挑刺者 ≠ 改者）；② **不是全自主**——实测发生在用户真实任务里、skill-tempering 搭车监督，改在另一会话（人在环、多会话）。

**覆盖账本怎么织**（半自动 · 与 §2.2 同一道工序下移一层、**不可全机械化**）：① 机械种子(grep 分支 / `⏸` 决策点 / 自带护栏 / references) → ② 智能派生(接 profile：每条派生纪律一行 + toggle 轴叉乘，**粒度＝行为情境非行/文件**) → ③ 完整性 critic(还能进什么没织进格的情境) → ④ 结晶生长(真实任务撞到没格的情境就补格、网格不冻结)。**防爆**：等价划分 + pairwise 覆盖。**分形自洽**：原则层与覆盖层复用同一工序。

## §4 自包含 / 毕业焙烧

- **meta-skill 自身**零运行期外部依赖（吃自己第 7 条）。
- **毕业焙烧**：底座是**孵化炉层单一权威**；被测 skill 毕业（filter-repo 抽出）时，把它**命中的纪律 + 强制力档 + 盲区声明焙进它自己的 MAINTAINING**（封存一份冻结副本，像 recon directed-report 的封存语义），蜕掉对底座的引用 → **出厂 skill 100% 自包含**。共享只发生在孵化炉内。
- **结晶治理**：见 §2.2.5（pending + 留痕 + 复核）。

## §6 自我 dogfood（底座吃自己的狗粮）

`skill-tempering` 必须能**优化它自己**：拿它的 7 核 + 派生工序审它自身。

- 这是它的活体自检与最强 dogfood：**优化不了自己就是坏的**。
- 已天然成立：前几轮验证表明，底座对"skill 类型"正是用①自评不可信（不硬套、当场派生）、④确定性归工具判断归模型（核机械、派生智能）、完整性 critic（声明盲区不假装穷尽）在运作。
- **本轮落地**：用**短路径审计透镜**审 `skill-tempering` 自己（静态自检）。
- **后续**：长路径"拿自己当第一个 dogfood 对象"随长路径一起单独讨论。

## §7 设计依据（4 轮 workflow 的证据沉淀）

- **§7.1 两层洞见**：运行层（独有内容、必自包含、永不共享）vs 维护层（可共享）。优化机制整个住维护层，故不破自包含。
- **§7.2 对抗审查**（遗漏/cross-cutting/重叠/altitude）：原 9 条漏掉"人机权责+运行鲁棒性"半边；理论接地是元规则非对象级原则（否则自指 no-op）。
- **§7.3 按型验配**（代码分析/工作流/文档/对话）：型档案"枚举"不行；真风险是 universal 套错"实例化口味"。
- **§7.4 6 轴压测**（安全/ETL/配置/集成/评测/浏览器）：**轴也是开放集**（每个怪型冒新轴），且"一 skill 一轴读数"假设破裂（skill 是异质复合体）→ **没有固定脚手架能机械派生任意 skill 的 profile；枚举是跑步机**。
- **§7.5 先例验证**：方向 = 工业标准（ESLint recommended 小核 + 可组合 preset + plugin 开放扩展；design token 三层；Google 五元原则；OPA data 注入；WCAG 分档+部分符合声明）。借入 6 升级件（superpowers 失败形态 / mattpocock 机制层 / Biome 嗅探+三态 / Sentinel 强制力档 / RuboCop pending / WCAG 部分符合声明）。⚠️ 我们的"结晶"是唯一无成熟先例的部件——加治理闸。强于 linter 处 = 完整性 critic，保留。
- **§7.6 机制层 pass**（mattpocock）：通用核 8 → 7，A1+A2+C7 合并为「自评不可信」，最尖的误配（A1 套创意类造假闸）自此焊进原则的降级支、消失。

## 待评审 / 开放问题
1. `skill-tempering` 名字仍是工作名（淬炼/回火金属隐喻＝把已有的东西变韧；可换）。
2. 7 通用核的强制力档默认值（hard/soft/advisory）需第一次真实 dogfood 校准。
3. 结晶治理的"复核责任人"在单人维护场景如何落地（自审 + 留痕够不够）。
4. 短路径"baseline 双跑"在纯静态审计里成本是否划算，还是只长路径用。

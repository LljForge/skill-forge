# 设计：codebase-exploration 全面改造（跨栈 · 多能力 · 不易过时的活地图）

- **日期**：2026-06-21
- **状态**：已确认，待落地
- **类型**：现有 Skill 重构（`codebase-exploration` v0.5.0 → v0.6.0）
- **关系**：与 `module-brief` / `module-spec-baseline` 分工不变（互不依赖）；下游仍指向 `/module-brief`
- **灵感来源**：GitNexus 深挖（借其跨栈信号**知识目录**，不借 tree-sitter 重机器、不接其引擎）

---

## 1. 背景与动机

`codebase-exploration` 现状（v0.5.0）：单一能力「模块测绘」→ `docs/codebase-map.md`，刻意轻量（grep+LLM、零依赖、启发式、可能错），自我定位「**探索草稿·非权威**」。但它有四处不足，需「全面改造」而非小修：

1. **太绑 Java/Spring**：信号采集写死 Java（`^import com.<root>.`、`@Autowired`、`@MapperScan`、MyBatis XML、`@TableName`、DDL），其它栈跑不动。
2. **能力该扩、但要有闸**：不应只有「模块测绘」一种；要在**「慢变内容纪律」前提**下扩能力。
3. **不够准/不够深（高层意义上）**：模块切分与依赖判定仍是粗启发式；要更可靠的**高层**结构（但不下钻代码细节）。
4. **产物是「带日期的冻结草稿」、易过时**：要做成**不易过时的活地图**，既是**人接手项目的资产**，也是**AI 编程的架构指引**。

**反目标（明确不做）**：不接 GitNexus 当引擎、不变成 tree-sitter 重型工具、不扩成「调用链/影响面」那类深细节多能力套件、不下钻代码细节（细节委托 `/module-brief`）。

---

## 2. 目标与非目标

**目标**
- 去 Java 绑定 → **跨栈**（信号语言无关化）。
- **多能力探索框架**，每能力过「慢变内容纪律」准入闸。本轮纳入：**① 模块测绘（跨栈化）+ ⑤ 约定与横切机制**。
- 产物**不易过时**（慢变内容纪律为主 + 漂移自检兜底）。
- 产物**双受众**（人 + AI 编码 Agent），定位从「草稿」反转为「**活文档/资产/AI 指引**」。

**非目标 / 本轮排除或暂缓**
- **排除 impact（影响面/改动半径）**：本质随每次提交而变，违慢变纪律，不进静态地图（留 `/module-brief` 或临时查询）。
- **暂缓 ②③④**（入口与执行流 / 数据模型 / API 表面）：留作框架「未来能力槽」（加节不动骨架），本轮不建。
- 不接 GitNexus、不上 tree-sitter、不另起机器可读层。

## 4. 架构：共享基础设施 × N 能力（三层）

```
共享基础设施（一次建好，所有能力复用）
  · 信号配方层 Signal Recipe   跨栈 de-Java：每栈每能力声明要 grep 的「信号槽」填法
  · 慢变纪律闸 Slow-Change Gate 统一过滤 + 能力准入：只留抗重构结构，剔除易变细节
  · 产物契约 Artifact Contract 双受众（人+AI）、新鲜度戳、AGENTS.md 指针、定位=活文档

每个能力 = 自己的信号槽 + 自己的方法判定 + 自己的地图节（加节不动骨架）
  · 能力① 模块测绘（跨栈化）
  · 能力⑤ 约定与横切机制
  · 〔留槽〕②入口与执行流 / ③数据模型 / ④API 表面
```

**关键接缝**：信号配方层只回答「**这个栈里，每个信号槽该 grep 什么**」（机械）；测绘方法主干（1.A–1.D 智能判定）原样保留、只改成消费「配方产出的信号」而非写死的 Java import。**de-Java 只动配方层，方法主干与智能判定不被推翻**。

**数据流**：探测栈 → 载/派生配方 → 1.A 机械跑信号 grep → 1.B–D 智能成依赖矩阵 → 卡片/拓扑序/陷阱 + ⑤ 约定 → 慢变纪律闸过滤 → 写 `codebase-map.md` + AGENTS.md 指针 + 新鲜度戳。

## 5. 信号配方层（(iii) 混合 · 技术核心）

### 5.1 通用信号槽（① 与 ⑤ 共用）

| 信号槽 | 服务能力 | Java 参考填法（首份结晶） | 跨栈含义 |
|---|---|---|---|
| 模块单元 | ① | package 目录 | 该栈「候选模块边界」是什么（TS 特性目录 / Go package / Python package） |
| 导入依赖边 | ① | `import <root>.` | 跨文件/模块依赖怎么表达 |
| 依赖装配（强度） | ① | `@Autowired`/`@Resource`/构造注入 | 依赖怎么接线 → 定 Injected vs Import-only |
| 入口点 | ①⑤ | Controller / `@Scheduled` / main | 路由/CLI/定时/消费者/main |
| 数据访问 | ① | MyBatis / `@TableName` | ORM/repo → 表/实体映射（辅助锚定模块） |
| 自声明清单 | ① | Maven `<modules>` / `@ComponentScan` | 构建/模块声明（起骨架） |
| 横切件 | ⑤ | `@Aspect` / HandlerInterceptor | AOP / middleware / 拦截器 / 装饰器 / filter / guard |
| 约定锚点 | ⑤ | base 类 / 全局异常处理器 / SecurityConfig | 认证/错误/日志/配置/事务/缓存 在哪定义 |

### 5.2 三段式来源（混合）

1. **通用层（直接抄 GitNexus，已被 16 栈验证，用自己的话重表述）**
   - 入口点通用名模式：`^(main|init|bootstrap|start|run|setup|configure)$`、`^handle[A-Z]`、`^on[A-Z]`、`Handler$`、`Controller$`、`^(process|execute|perform|dispatch|trigger|fire|emit)[A-Z]`。
   - 噪音/工具名模式（降权，非入口）：`^(get|set|is|has|can|should)[A-Z]`、`^_`、`^(format|parse|validate|convert|transform)`、`^(log|debug|error|warn)$`、`Helper$`/`Util(s)?$`。
2. **蒸馏框架约定速查表（从 GitNexus `framework-detection.ts` ~13 生态 ~70 条提炼，作派生种子）**
   - 「栈→框架→文件约定」：Next.js `pages/`·`app/page`·`api/route` / Expo `app/+api` / Express·MVC `routes/`·`controllers/`·`handlers/` / Django `views.py`·`urls.py` / FastAPI·Flask routers / Spring `*Controller`·`*Service` / Ktor routing·plugin / ASP.NET controller·middleware·SignalR·Blazor / Go `main.go`·handlers·routes / Rust `bin`·main·handlers / Laravel `routes/`·controllers·jobs·listeners·models·middleware·providers / Rails / SwiftUI·UIKit·coordinator / Flutter `main`·screen·widget·state。
   - 横切件标记（喂 ⑤）：aspnet-middleware / laravel-middleware / ktor-plugin / `@Aspect` / 全局异常处理器 等。
3. **长尾运行时派生**：探测到栈后照槽模型 + 速查表**起步、现场补**；某栈跑顺了像 Java 一样**结晶**进 spec 成第二份参考。

### 5.3 栈探测与许可证护栏

- **栈探测**：按 manifest（`pom.xml`/`build.gradle` → Java；`package.json`+`tsconfig` → TS；`pyproject`/`requirements` → Python；`go.mod` → Go；…），多栈仓库各跑各配方再合并。
- **许可证护栏**：GitNexus 为 PolyForm Noncommercial。**不 vendoring 其代码、不依赖它**；借的是「框架怎么摆文件」这类公共事实约定，**用自己的话重表述**为紧凑参考——对「地图当资产复用」干净。
- **机械/智能分工铁律（守 v0.3.x 血泪）**：配方只定「grep 什么」（机械，可借 Bash）；归并/强度/反向异味仍是 agent 现场智能判定，**不得借「派生配方」之名脚本化**。

## 6. 能力① 模块测绘（跨栈化）

方法主干**沿用现 v0.5.0 的两步**（依赖矩阵采集 1.A–1.D + 业务语义提取），只把 1.A 的「机械采集」改为**跑配方的信号槽 grep**，1.B–D 智能判定不变（消费配方产出而非写死 Java import）。

- **跨栈模块单元**：模块边界由配方的「模块单元 + 依赖装配 + 数据访问」交叉判，非 `ls 包`。
- **保留 Java 深度（CHANGELOG 血泪全留为 Java 配方/陷阱）**：扁平分层同包补扫、锚 PO 交叉判、伞形包拆够、审计层伪装、同表多 PO、依赖方向必出依赖矩阵（不望名生义）、触发器写入必 grep 核实。
- 输出：模块总览 + 模块卡片（边界/职责/入口/依赖摘要/置信），伞形包拆到 `/module-brief` 单次可消化粒度。

---

## 7. 能力⑤ 约定与横切机制

现「全局基础设施」节的升级——从「列横切件」扩成「**加东西要守什么规矩**」（最高 AI 指引价值）。

- **测绘什么（慢变）**：① 横切件（AOP/middleware/拦截器/装饰器/guard/filter——「请求先过谁」）；② 项目约定（认证、错误处理、日志、配置、事务、缓存——每条**在哪定义 + 怎么遵守**）。
- **信号槽**：横切件槽 + 约定锚点槽，种子来自 §5.2 框架目录 + 现有「共享度抓横切」法（被多文件引用 = 横切）。
- **慢变纪律**：只记**机制 + 定义点（代表锚点）+ 怎么遵守（一句）**，**不穷举使用点**。表达为**可执行的约定规则**，例：
  > `认证 — 所有 Controller 经 JwtAuthFilter（security/JwtAuthFilter）；加新接口默认受保护，放行须进 SecurityConfig 白名单。`
- **校准（延续灵魂）**：约定确认不了标**存疑**，绝不脑补；写入路径必 grep 核实（trigger/AOP/拦截器）。⑤「不当业务模块」，与 ① 业务卡片分开。

## 8. 慢变纪律闸（产物过滤器 + 能力准入闸）

| 进（抗重构的慢变结构） | 出（易变细节，交 `/module-brief`） |
|---|---|
| 模块边界与职责、分层 | 行号、精确文件/类计数 |
| 依赖**形状与方向** | 穷举 PO/类清单 |
| 入口约定、横切约定 | 随提交而变的度量 |
| 核心实体↔模块归属 | 具体实现细节 |
| 真陷阱（同表多映射 / hub / 触发器隐式耦合） | — |

度量数字处理沿用现规则（翻成读者结论、全文 ≤ ~8 处）。**准入**：候选能力须能在「进」栏粒度表达，否则不收（impact 即被此排除）。

## 11. 已知妥协与风险

- **跨栈派生覆盖不确定**：未结晶栈靠运行时派生 + 速查表，弱模型可能漂移；缓解=速查表种子 + 逐步结晶 + 校准存疑。
- **定位反转 vs 「校准优先」张力**：用「慢变闸抬高权威度 + 置信标记收窄到模糊边界」化解；若实跑发现 agent 滥用「活文档」抹掉该标的低置信，再收紧。
- **依赖矩阵代表锚点仍触及类名**（轻度「代码细节」）：作导航代表、非穷举/非深析，慢变可接受。
- **⑤ 约定确认成本**：约定散落时 grep 可能漏；靠校准存疑兜底，不脑补。

## 12. 与邻居 Skill 关系（不变）

| Skill | 性质 | 何时用 |
|---|---|---|
| **codebase-exploration**（本） | 广、快、轻量、跨栈、慢变活地图 | 接手项目建全局认知 / 给 AI 当架构指引 |
| module-brief | 选一模块读进上下文产需求+设计 | 探索后深入读懂某模块（细节归这） |
| module-spec-baseline | 逆向成 openspec 规范基线 | 为某模块引入 openspec 治理 |

三者互不依赖契约不变；下游仍 `/module-brief <模块>`。

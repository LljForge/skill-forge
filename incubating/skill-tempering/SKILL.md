# skill-tempering（淬炼 · skill 优化 meta-skill）

一个 **meta-skill**：拿一份**现成** skill，把它**优化 / 成熟**到本仓标准（回火淬炼——实测到位、去脆性、变韧）。它住**维护层**——是"作用于"一个 skill 的过程，不是 skill 运行时"依赖"的库；产物是"一个更好的、仍然自包含的 skill"。

本文件只持**脊柱**：两路径流程图、7 通用核（只名 + 一句 + 指向权威全文的指针）、派生工序与短路径入口、暂停点、全局护栏。**判据正文一律不在脊柱复述**——7 核全文唯一在 `references/principles.md`，派生工序唯一在 `references/derivation-protocol.md`，短路径细则与模板唯一在 `references/audit-lens.md`，理论依据唯一在 `theory-foundation.md`。指针只点该 reference 里**非默认的具体动作**，不是空转的"详见 X"。

## 核心流程：两条路径（共用同一底座，短 ⊂ 长）

```
                    ┌─ 一份现成 skill ─┐
                    │                  │
   ┌────────────────┴──────────┐   ┌───┴──────────────────────────────┐
   │ 短路径 · 审计透镜（本轮已实现）│   │ 长路径 · dogfood 成熟（后续·本轮不实现）│
   │ 静态体检、单会话、便宜       │   │ 真实任务实测、跨会话、贵            │
   │ 归型 → 跑 7 核 + 派生纪律    │   │ 实测 → 三分诊 → 处置 → 验证 → 沉淀  │
   │      → 完整性 critic        │   │ （细则后续单独讨论，本轮不展开、       │
   │      → 产改进清单 + 盲区声明  │   │   暂无独立文件；要点见设计稿 §3.2）   │
   └────────────────┬──────────┘   └──────────────────────────────────┘
                    ⏸ 产出改进清单后停、交用户（不自动改被测 skill）
```

- **短路径 = 长路径的"验证 / 审计"步独立拎出来跑**；长路径含短路径。
- **本轮只实现短路径**。**长路径是后续工作、本轮不实现、不展开**——它没有独立文件，别去找 / 别建悬空指针；若要看已啃下的设计要点，去**设计稿 §3.2**（foothold，唯一权威）。

## 7 通用核（机制层 · 任何 skill 都先逐条过）

> **只列名 + 一句**。每条的原则全文 / 触发条件 / 审计问句 / 强制力档默认，**唯一**在 `references/principles.md` 对应锚 `#N`——拿那里的**审计问句**去问被测 skill，按其**触发条件**判管不管它、按**强制力档**定违规处置力度。脊柱不复述。

1. **自评不可信**——产出方与裁判方必须分开；无外部真值就诚实标低置信 / 交人，绝不自评放行。判据全文与审计问句去 `references/principles.md` 锚 `#1`。
2. **完成判据**——显式、可判、带死上限的"做到这儿就停"；无界循环或无判据即缺陷。去 `references/principles.md` 锚 `#2`。
3. **快乐路径之外**——为可预见的失败 / 前置缺失 / 能力不具备各定义显式降级或退出，不默认一切顺利。去 `references/principles.md` 锚 `#3`。
4. **确定性归工具、判断归模型**——能机械确定的别让模型猜，需判断的别硬编成死表（双向）。去 `references/principles.md` 锚 `#4`。
5. **单一权威源 · 守薄**——每条判据 / 清单只一个权威家、禁复述；抗蔓延 / 沉积 / 空转 / 重复四类熵增。去 `references/principles.md` 锚 `#5`。
6. **描述即激活闸**——name + description 写成"何时用"触发器（正向触发词 + 负向边界），非"本 skill 做 X"摘要。去 `references/principles.md` 锚 `#6`。
7. **自包含 · 可移植**——零运行期外部 skill / 脚本 / 跨会话依赖；改一个 skill 必从它自身长出。去 `references/principles.md` 锚 `#7`（本核是**唯一硬约束**，连改本 meta-skill 自己时也不可 override）。

> 每条"理论骨"展开一句结论 + 出处，唯一在 `theory-foundation.md`（按 `#1`–`#7` 对位）——要查"这条凭什么成立"去那里，**不在 principles.md 与本脊柱重述理论**。

## 派生工序（7 核之外的"额外适用纪律"，现场长出）

7 核之上，被测 skill 因形态不同还该套上的纪律——**不靠枚举类型查表，靠一道现场判断工序长出**：归型（嗅可观测事实 + 看失败形态）→ 查种子库挂条件纪律 / 缺了当场派生 → 完整性 critic 揪种子没问到的命脉 → 盲区声明。完整六步、种子库表、结晶治理闸，唯一在 `references/derivation-protocol.md`——短路径要走的是其中 **①归型 / ②种子库 / ③完整性 critic / ⑥盲区声明**（④实证双跑属长路径、⑤结晶治理属维护层，去那里看分界标注）。

## 短路径入口（本轮唯一可跑的路径）

执行短路径，**不在脊柱编排步骤**，去 `references/audit-lens.md`——那里有完整的**七步静态体检流程** + **反"过度报缺口"护栏**（刹车：宁可少报准报，不可满屏报缺口）+ **改进清单模板** + **结构化部分符合声明模板**。脊柱只交代它的输入与产物：输入一份现成 skill 的文本载体（只读不跑），产出一份按强制力档分级（hard / soft / advisory）的改进清单 + 一份"永不宣称全覆盖"的部分符合声明。

## ⏸ 暂停点

**产出改进清单（+ 部分符合声明）后停下，交给用户。** 本 skill **不自动去改被测 skill**——它只诊断、列改进项；改与不改、怎么改由用户定。这是短路径唯一的硬暂停。

## 全局护栏

> 这些是跑 skill-tempering 时的运行约束。**改本 skill 自己**时的治理（防膨胀 / 单一权威源 / 结晶闸落地）属维护宪法、不随运行载入，见 [`MAINTAINING.md`](MAINTAINING.md)。

- ✅ **自包含是唯一硬约束（7 核 #7）**——本 meta-skill 自身零运行期外部依赖（吃自己第 7 条）；它住维护层、不进被测 skill 的运行期，故 structurally 不破被测 skill 的自包含。
- ✅ **守薄（7 核 #5）**——判据全去 references 读，脊柱只路由不复述；指针只点非默认的具体动作，不堆砌。
- ❌ **不自动触发**——重型 meta 动作，只由用户显式调用（同 description 负向边界）。
- ✅ **诚实标盲区（7 核 #1）**——短路径纯静态、未实测的派生纪律一律标"静态推定、未实证"，绝不冒充已验证；没把握就降 advisory 或写进部分符合声明"待确认"，不报成违规。

# 通用核：7 条机制层原则（权威锚）

> 本文件是 `skill-tempering` 7 条机制层通用核的**唯一权威定义处**——编号 `#1`–`#7`、各自名称、各自`强制力档`默认值只在此处落地。后续 `derivation-protocol.md` / `audit-lens.md` / `SKILL.md` / `theory-foundation.md` 一律引用本文件的 `#1`–`#7`，**全仓不再二次定义**（守 #5：单一权威源）。
>
> **怎么读**：任何一份被淬炼的 skill 都先逐条过这 7 核——拿每条的`审计问句`去问被测 skill，按`触发条件`判它该不该被这条管，按`强制力档`定违规的处置力度。7 核之外的"额外适用纪律"不在此枚举，由派生工序现场长出（见 `derivation-protocol.md`）。

---

## 两个"硬"别混（先读这条，否则下文每条的"硬"都会读歪）

本文件出现两处"硬"，**正交、互不蕴含**：

- **强制力档 `hard`**：审计纪律层的取值。指"本次审计中，被测 skill 在这条上违规 → 必改、不可放过"。它针对**被测 skill**，逐次审计可现场覆盖（覆盖须留痕）。
- **「唯一硬约束」**：设计宪法层的标签，只贴在 #7（自包含·可移植）。指"连 `skill-tempering` 作者改它自己时，都不可 override 自包含"。它针对**meta 层设计**，是宪法级、不可覆盖。

前者是审计纪律、后者是设计宪法。一条核的强制力档是 `hard`，不等于它是「唯一硬约束」；唯有 #7 同时是 `hard` 且唯一不可 override。

---

## 强制力档取值

`advisory`（建议）/ `soft`（默认改、可现场 override 并留痕）/ `hard`（违规必改）。各核的默认档见下表，可在单次审计中现场覆盖，覆盖须留痕。

| 编号 | 名称 | 强制力档默认 |
|---|---|---|
| #1 | 自评不可信 | `hard` |
| #2 | 完成判据 | `soft` |
| #3 | 快乐路径之外 | `soft` |

| #5 | 单一权威源·守薄 | `soft` |

## #1 自评不可信（generation ≠ judgment）

- **原则**：被淬炼对象对自己输出的正确性没有可靠的内部信号；产出方与裁判方必须分开。**承诺的形态决定验证的形态**——
  - 承诺一个点真值 → 用外部来源核那个点；
  - 承诺完整/穷尽 → 核全集而非抽样自夸；
  - **无外部真值可核 → 退到诚实地标低置信 / 标"待确认"、把判断权交人，绝不自评放行、不假装已完成**。
- **触发条件**：流程里**任何一步要采信被测对象对自身输出的断言**（"我做对了""我覆盖全了""这就是答案"）时触发。
- **理论骨**：内省式自纠在缺外部信号时不可靠，常越纠越差——见 `../theory-foundation.md` #1。
- **审计问句**：它在哪一步信了被测对象对自己的断言？那一步有外部真值可核吗——有就该接外部闸去核、无就该退到诚实标注并交人，而不是自评放行。
- **强制力档默认**：`hard`。
- **收编**：本核已合并原 maker≠checker、认识论诚实、完整性闸三条——三者是同一机制（产出与裁判分离）在不同承诺形态下的分支，不再单列。

## #2 完成判据（completion criterion）

- **原则**：显式、可判、带上限地界定"做到这儿就停"。凡含自检 / 迭代 / 重试循环者，必须给一个能机械判定的死上限；无界循环、或根本没有"算完"的判据，都是缺陷（够用即止，反对为追求完美而空转）。
- **触发条件**：流程里**存在任何循环、重试、自检迭代，或需要回答"什么时候算做完了"**时触发。
- **理论骨**：留控制权须给迭代上限，无界自我循环是已知反模式——见 `../theory-foundation.md` #2。
- **审计问句**：它界定了"到什么程度算完成"吗，这判据是可机械判定的还是含糊的？它的每个循环 / 重试都有死上限吗，到顶时怎么收场？
- **强制力档默认**：`soft`。

## #3 快乐路径之外（failure / absence path）

- **原则**：为可预见的失败、前置缺失、能力不具备各定义显式行为（降级、或优雅退出并说明），不默认"一切都会顺利"。
- **触发条件**：流程**依赖任何可能不成立的外部条件或前置**（所需输入可能缺失 / 异常、所需能力可能不具备、自检可能不收敛）时触发。
- **理论骨**：鲁棒性要求显式覆盖失败与缺失分支，而非只规定顺利情形——见 `../theory-foundation.md` #3。
- **审计问句**：当它依赖的外部条件不成立时（前置缺失、输入异常、自检不收敛），它是崩在只写了顺利情形的路径上，还是有显式的降级 / 退出分支？这些分支写明了吗、可执行吗？
- **强制力档默认**：`soft`。

## #4 确定性归工具、判断归模型

- **原则**：能机械确定的事别让被测对象去猜（交给确定性工具去算 / 去核）；真正需要判断的事别硬编码成死规则（留给模型现场判）。双向：既反对该用工具时让模型猜，也反对该让模型判时塞进僵死的查表。
- **触发条件**：流程里**出现"由谁来定这件事"的分工抉择**——某一步既可交确定性工具、也可交模型时触发。
- **理论骨**：可计算的事项归确定性调控、需推断的事项归推断式调控，两者错配即缺陷——见 `../theory-foundation.md` #4。
- **审计问句**：它有没有让模型去做本可机械确定、本该交工具的事？反过来，它有没有把本需现场判断的事硬编码成一张写死的表，剥夺了该有的判断空间？
- **强制力档默认**：`soft`。

## #5 单一权威源·守薄

- **原则**：每一条判据 / 清单 / 动作只有一个权威出处，别处只引用不复述；持续对抗四类熵增——蔓延（无节制地长）、沉积（过时残留不清）、空转（写了等于没写）、重复（同一件事多处各写一份）。
- **触发条件**：**任何时候**触发——这是贯穿性纪律；尤其在被测对象有多个文件 / 多处清单、或经历过多轮增改时最该查。
- **理论骨**：单一权威源是抗冗余、抗漂移的结构地基——见 `../theory-foundation.md` #5。
- **审计问句**：同一条判据 / 清单 / 动作，是不是在两处以上都各写了一份（而非一处定义、别处引用）？有没有已经过时却没清掉的残留，或写了却起不到任何约束作用的空转条目？
- **强制力档默认**：`soft`。

## #6 描述即激活闸

- **原则**：被测对象的名称 + 描述，决定它能否在对的时机被唤起。描述要写成"何时该用它"的触发器——含正向触发词 + 负向边界 / 反例分流——而不是"它做什么"的功能摘要。倾向是写得过窄而欠触发，要校到对的时机能命中、错的时机被边界挡住。
- **触发条件**：**任何会被自动 / 半自动唤起的被测对象**都触发——只要它的被唤起依赖一段自我描述，这条就管。
- **理论骨**：描述是触发器而非摘要，欠触发是常见失配——见 `../theory-foundation.md` #6。
- **审计问句**：这段描述能让它在该出场的时机被唤起吗——它写的是"何时用它"还是只在概括"它做什么"？它有没有划出负向边界 / 反例，把不该出场的相邻时机挡在外面？
- **强制力档默认**：`hard`。

## #7 自包含·可移植

- **原则**：零运行期外部依赖——运行时不依赖别的 skill、外部脚本、跨会话状态；并且改一个 skill 时，所依据的纪律必须能从它自身长出，不借外部方法论。一份合格的被测对象，搬到任意环境都能独立跑起来。
- **触发条件**：**任何时候**触发——这是可移植性的底线纪律；尤其在被测对象引用了外部资源、或其改动依据来自别处时最该查。
- **理论骨**：可移植 / 可分发要求运行期零外部依赖，是分发模型的地基——见 `../theory-foundation.md` #7。
- **审计问句**：它在运行期有没有依赖别的 skill / 外部脚本 / 跨会话状态？改它时，所凭的纪律是从它自身长出的，还是借了某套外部方法论？
- **强制力档默认**：`hard`·唯一硬约束（连改本机制自己时也不可 override，见本文「两个『硬』别混」节）。

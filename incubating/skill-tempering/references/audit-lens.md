# 短路径 · 审计透镜：静态体检细则 + 模板

> 本文件是 `skill-tempering` **短路径（审计透镜）** 的执行细则。它**只编排流程、只给填空模板**——判据归 `principles.md`（七核 `#1`–`#7` 及各自审计问句）、归型 / 派生 / 完整性 critic / 盲区声明归 `derivation-protocol.md`（① / ②种子库 / ③ / ⑥）。本文件**只引锚、不复述其内容**（守 `principles.md` #5：单一权威源）；任何处复述上游判据即违铁律。
>
> **短路径是什么**：拿一份现成 skill 做一次**纯静态体检**——读它、归它的型、拿七核审计问句加派生条件纪律逐条判、过一遍完整性 critic，产出一份分级改进清单加一份结构化部分符合声明。**单会话、便宜。** 它 = 长路径（`dogfood-protocol.md`）里"验证 / 审计"那一步独立拎出来跑。

---

## 短 / 长分界（先读这条，划清边界）

**短路径纯静态，不执行被测 skill。** 它只**读取**被测 skill 的文本载体（脊柱 + references + 描述等），靠 derivation ①–③ 的嗅探、七核审计问句、完整性 critic 产清单——**全程不跑被测 skill、不喂真实任务、不开第二会话**。

凡涉及**实际运行**的环节都属长路径、不在本文件：

- **baseline 双跑**（with / without 对照、实测纪律灵不灵）＝ 长路径（derivation ④ 已标"baseline 双跑属长路径"；细则见 `dogfood-protocol.md`）。
- **实测会话 / 打磨会话 / 三分诊 / 回填覆盖账本** ＝ 长路径。

故短路径里凡靠归型嗅出、未经实测的派生纪律，一律是**静态推定**——必须按 derivation ⑥ 在部分符合声明里标"静态推定、未实证"，**绝不冒充已验证**（#1：自评不可信，对本工序自己派生的纪律同样适用）。

---

## 执行步（七步静态体检流程）

> 每一步只说"做什么 + 调上游哪一锚"，判据正文一律去对应锚里读。

### 第 1 步 · 读 skill（纯静态）

通读被测 skill 的全部文本载体：脊柱（SKILL.md）、references、name + description、自带的护栏 / 决策点 / 清单。**只读不跑。** 这一步只为后续归型与逐条判积累可观测事实，不下任何结论。

### 第 2 步 · 归型（走 derivation ①）

按 `derivation-protocol.md` **①（嗅 + 失败形态）** 归型：A 路嗅可观测事实（五项事实清单见 derivation ① A，此处仅举要、非穷列），B 路看失败形态。

> 短路径限制：① 的 B 路"跑 baseline 看缺它时怎么坏"在短路径里**只做静态推断**——基于文本推测最可能的失败形态，不实跑取证。推断出的失败形态一律视作静态推定，进第 7 步盲区声明。

### 第 3 步 · 逐条判（七核审计问句 + 派生条件纪律，各判三态）

两批逐条判，**判据全去上游读、本文件不抄**：

- **七核**：拿 `principles.md` `#1`–`#7` 各自的**审计问句**去问被测 skill，先按各条**触发条件**确认这条管不管它，再按其**强制力档默认**定档。
- **派生条件纪律**：把第 2 步嗅到的事实 / 失败形态喂进 `derivation-protocol.md` **②种子库**，命中触发就挂上对应纪律、带其强制力档。

每一条判出**三态之一**：

| 三态 | 含义 | 进清单与否 |
|---|---|---|
| **通过** | 这条原则 / 纪律实际适用，且被测 skill 满足 | 不进清单 |
| **违规** | 这条实际适用、被测 skill 不满足 → 按其 `强制力档` 定处置力度（档义见 `principles.md` 文件头） | **进清单** |
| **建议（advisory）** | 这条对被测 skill 只沾边、非命脉（边缘适用） | 进清单**但明标"可选"**，不逼改 |

> 强制力档三档（`advisory` / `soft` / `hard`）的取值定义在 `principles.md` 文件头「强制力档取值」，本文件不复述；现场覆盖须留痕，留痕规则同 `principles.md`。

### 第 4 步 · 反"过度报缺口"护栏（关键，见下方独立章节）

第 3 步逐条判时**全程套用**下方「反『过度报缺口』护栏」——它是本短路径的刹车，决定哪些判出的问题**该报、哪些不报或只标 advisory**。这一步不是流程的独立一环，而是贯穿第 3 步每一条判的过滤纪律，单列强调因其是验收关键。

### 第 5 步 · 完整性 critic（走 derivation ③）

按 `derivation-protocol.md` **③** 显式追问一句"这个 skill 有什么命脉，是七核加种子库一条都没问到的？"。critic 揪出的疑似命脉：

- 当场派生一条新维度纪律（拟名 + 触发 + 暂定强制力档），挂上本次审计、按第 3 步三态判它。
- critic 产出**不可只靠内省自封**（#1）——短路径无法实证，故新派生维度一律进第 7 步盲区声明标"待验证"。
- **结晶治理（derivation ⑤）属维护层、不在短路径产出流程里**：当场派生的维度只对本次审计生效，是否沉回种子库由 ⑤ 的 `pending + 留痕 + 复核` 闸另行裁决，本文件不复述该闸。

### 第 6 步 · 产改进清单（用下方模板）

把第 3–5 步判出的所有**违规**与**建议（advisory）**条目，填进下方「改进清单模板」。**通过项不入清单。** 清单按强制力档分级排列（hard → soft → advisory），advisory 项一律明标"可选"。

### 第 7 步 · 产结构化部分符合声明（走 derivation ⑥，用下方模板）

按 `derivation-protocol.md` **⑥** 产一份结构化部分符合声明，填进下方模板。**永不宣称"已覆盖所有 skill 类型 / 所有适用纪律"。** 凡静态推定未实证的派生纪律（第 2 步推断的失败形态、第 5 步 critic 派生的新维度），都在此各记一行。

## 反"过度报缺口"护栏（刹车 · 验收关键）

> 依 Anthropic skill 创作 best-practices 的取向：清单要**精准、可执行**，不是越长越好。满屏过度报缺口会淹没真问题、逼出不必要的改动，比漏报更伤可用性。

逐条判时套这三条刹车：

1. **只报违反"实际适用"原则的问题。** 一条原则 / 纪律必须先过它自己的**触发条件**（见 `principles.md` 各核「触发条件」、`derivation-protocol.md` ②种子库「触发」列）确认**确实适用于被测 skill**，才可能进清单。触发条件不成立 = 这条不管它 = **判"通过"或根本不判，绝不报**。

2. **边缘适用 / 不适用的，不报或只标 advisory。** 一条只跟被测 skill 沾边、非其命脉的纪律（判定"命脉还是沾边"是判断活、归模型现场判，依 derivation ② 末「边缘适用 → advisory」），**降为 advisory，绝不硬扣成 hard / soft 去逼改**；完全不适用的，**直接不报**。

3. **advisory 必须明标"可选"。** 凡 advisory 条目，在清单的`强制力档`列明确写 `advisory`、并在`建议改法`里点明"可选 / 非必改"，让接收者一眼分清"必须改"与"可以考虑"。

> 一句话：**宁可少报准报，不可满屏报缺口。** 没把握某条实际适用时，降 advisory 或写进盲区声明"待确认"，而不是报成违规（这本身正是 #1：无外部真值时退到诚实标注、不自评放行）。

## 模板一 · 改进清单（纯填空骨架）

> 这是填空骨架，**不是判据清单**。每条该报什么、判成几档、命脉还是沾边——判据全在 `principles.md` / `derivation-protocol.md`，本模板只给四列填空结构。只填**违规**与**建议（advisory）**项；通过项不填。按强制力档 hard → soft → advisory 排列。

```
## 改进清单 · <被测 skill 名> · <日期>

### 归型摘要（一句话，填第 2 步嗅到的关键事实 + 推断失败形态）
- <这个 skill 嗅到的命脉事实（按 derivation ① A 嗅，静态推定）+ 推断失败形态（按 ① B）>

### 清单条目

| # | 原则锚 | 强制力档 | 现场证据 | 建议改法 |
|---|---|---|---|---|
| 1 | <#N 或 派生纪律名> | <hard / soft / advisory> | <file:line 或 具体引文> | <怎么改；若 advisory 须写"可选"> |
| 2 | <…> | <…> | <…> | <…> |
| … | | | | |
```

**四列填什么（只说填空口径，判据去上游读）：**

- **原则锚**：违的是哪条——七核填 `#1`–`#7`（来自 `principles.md`），派生纪律填其名（来自 `derivation-protocol.md` ②种子库，或第 5 步 critic 当场派生的拟名）。
- **强制力档**：`hard` / `soft` / `advisory` 三选一（取值定义见 `principles.md` 文件头；现场覆盖默认档须在此条留痕注明）。
- **现场证据**：**可定位的客观锚**——`file:line`，或一段具体引文。**禁止只写"感觉不对""可能有问题"这类无证据断言**（无证据 = 退盲区声明"待确认"，不进本清单）。
- **建议改法**：可执行的改法一句话；`advisory` 条目此列**必须含"可选 / 非必改"字样**。

## 模板二 · 结构化部分符合声明（纯填空骨架）

> 与 `derivation-protocol.md` ⑥ 同源，本模板只给可填的骨架三列，判据 / 何为"部分符合"的语义去 ⑥ 读，本文件不复述。**每次短路径审计都附此声明，永不宣称全覆盖。**

```
## 结构化部分符合声明 · <被测 skill 名> · <日期>

| 未覆盖项 | 理由 | 责任人 |
|---|---|---|
| <这次没审到 / 没把握的命脉维度> | <为何没覆盖：种子没问到 / 短路径未实证(静态推定) / 超出本次范围> | <谁来跟进> |
| <…> | <…> | <…> |
```

**三列填什么（口径，语义去 ⑥ 读）：**

- **未覆盖项**：本次审计没审到或没把握的命脉维度。**短路径必填项**——第 2 步静态推断的失败形态、第 5 步 critic 派生的新维度，凡未实证者各记一行。
- **理由**：为何没覆盖。短路径最常见理由是"**静态推定、未实证**"（baseline 双跑属长路径，见上方短 / 长分界）。
- **责任人**：谁来跟进（可指向下一次审计 / 长路径 dogfood / 结晶治理 ⑤）。

---

> 自洽性回望：短路径正是七核在跑自己——#4（七核问句机械化、归型与命脉判断留模型）、#1（不自评放行、未实证就标盲区、降 advisory 不硬套）、#5（只引上游锚不复述、声明盲区不假装穷尽）。它审不了别的 skill、也审不了自己，即是坏的。

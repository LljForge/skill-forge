# 行为维度登记表(易漏维度优先)

本表是 module-spec-baseline「该覆盖哪些外部可观察行为维度」的**单一登记处**,刻意**独立于技术栈策略包**(web/orm/rpc):栈包只管"在某栈里**怎么找**",本表管"**找什么**"。新增一类维度在这里加一条 + 配套栈线索,不动既有发现门。

## 用法与边界

- 本表**只登记模型系统性容易漏的维度**。模型自己就会找的(端点的成功返回、CRUD、状态机迁移、字段约束)**不在此列**,默认已覆盖——契合"只补模型导不出的"原则,别把表堆成大而全。
- 判别底线(所有维度通用):一条维度该不该写进 spec,过**行为优先**测试——"一个只能从外部调用、看不到源码的人,能不能观察到?"能 → 是契约,登记;只有读源码才知道 → 不是,留 scratchpad 取证。
- 接入说明:本表是登记/契约;各分析 agent 对它的**实际消费分批接入**(横切契约、非 HTTP 入口),接入前这些维度按"先空着/标 N/A"处理,不强造。
- 栈线索一律给**通用示例**(以 Spring/Java 举例仅为说明),本表**不绑定任何具体项目**;某栈的精确召回写法归该栈策略包。

---

## A. 触发入口(非 HTTP)

> 易漏共因:发现习惯绕着 HTTP 端点转,**没有 HTTP 入口的行为整类看不见**。这类要从"谁会自动/异步触发"反向找。

### A1 · 定时/调度
- 是什么:无请求驱动、按时钟自动跑的行为。
- 可观察:到点数据被同步 / 账单被生成 / 通知被发出。
- 记在:scratchpad「非 HTTP 入口」。
- 栈线索:注解式定时(如 `@Scheduled`)、外部调度器 + cron 配置、Quartz/xxl-job。

### A2 · 消息队列(入站消费 + 出站发布)
- 是什么:消费 MQ 消息驱动的处理;以及"操作成功后向 MQ 发消息推动下游"。
- 可观察:消息到达后下游状态变化;**"接口返回成功 ≠ 下游已就绪"**(异步最终一致)。
- 记在:scratchpad「非 HTTP 入口」(消费)/「横切契约·副作用」(发布)。
- 栈线索:消费监听注解(如 `@RabbitListener`/`@KafkaListener`)、发送模板(如 `RabbitTemplate.convertAndSend`)。

### A3 · 启动钩子
- 是什么:应用启动即触发的初始化行为(仅当其有外部可观察后果时才登记)。
- 可观察:重启后自动跑一轮同步/预热等。
- 栈线索:`CommandLineRunner`/`ApplicationRunner`/`@PostConstruct`(多数只是建线程池/读配置,无可观察行为 → 不登记)。

## B. 横切契约

> 易漏共因:**跨多个端点、长在端点代码之外**,只读单个端点方法看不到。要单独拎出来扫,且**必须深读到真身**(别看到注解就臆测,守卫可能是空壳)。

### B1 · 事务边界与回滚/补偿
- 可观察:一组写操作部分失败时,是**整批回滚**还是留下脏数据;失败后是否有补偿/重试自愈。
- 栈线索:`@Transactional(rollbackFor=...)`、编程式事务、补偿端点、重试循环 + 失败重投。

### B2 · 审计/留痕副作用
- 可观察:每次操作/外呼是否留下可查的痕迹(谁、何时、入参出参、耗时、状态码)。
- 记在:scratchpad「横切契约·副作用」。
- 栈线索:入站操作日志切面(如 `@Around` 写动作日志)、出站调用级日志(每笔外呼落日志表)。

### B3 · 配置开关/凭据 + 环境差异
- 可观察:某行为是否发生,取决于配置开关(开/关行为分叉);不同环境/profile 行为不同(如某 cron 仅特定环境跑)。
- 栈线索:`@Value`/`@ConfigurationProperties` 后接分支、`.enabled` 开关、多 profile 配置差异(注意:这类契约**不在源码 AST 里**,需读配置文件)。

### B4 ·（已排除）出站签名/令牌
> **不进行为契约、逆向时不扫不写**:它只对**下游**可观察(本系统的调用方根本看不到我们怎么对下游签名),且离不开机制词("加密/令牌/派生"),本质是**集成实现**,而非本系统对其消费者的外部可观察行为。集成方若真需此约束,属对接文档、不属行为基线。
> 通用口径:横切契约**只记可观察后果,不记机制**——记「失败后数据是否一致」不记「删后插/异步线程」,记「记录可被查询到」不记日志表名。

### B5 · 鉴权/访问控制前置条件
- 可观察:端点是否需要有效身份才能访问;无凭据/过期返回什么(如 401);哪些路径豁免。
- **注意**:这是**黑盒可复现的端点前置契约**,不是"代码安全审计"——别因 skill"不做安全"把它一起排掉。**且必须深读到真身**:过滤器可能被禁用/空壳,照配置臆测会编造不存在的契约。
- 栈线索:安全过滤器链 + 豁免名单 + 未授权应答;令牌有效期/刷新机制。

### B6 · 缓存与读一致性
- 可观察:写后立即读是否能读到新值,还是有"读旧值"窗口(缓存未失效)。
- 栈线索:声明式缓存(如 `@Cacheable` 却无 `@CacheEvict`)、命令式缓存读写。

### B7 · 并发上限 + 幂等/去重
- 可观察:并发请求的具体上限/拒绝策略(超限被拒/降级);重复提交的后果(去重不重复生成 vs 报错)。
- 栈线索:信号量/线程池拒绝策略、分布式锁、去重键 + 唯一约束、`on duplicate key`。

## C. 契约形态(易被一句"返回成功/失败"带过)

### C1 · 错误响应契约 + 成功响应信封
- 可观察:出错返回什么码/状态/错误体;成功报文的真实信封结构(字段名、分页缺省)。调用方据此分支、解析报文——这是契约的另一半。
- 栈线索:全局异常处理器(状态码映射)、统一响应信封类、错误码枚举。

### C2 · 文件格式列契约 + 时区/国际化
- 可观察:导入导出文件的列序/表头/必填(变一个对接就破);时间字段的统一格式/时区。
- 栈线索:Excel 列注解(如 `@ExcelProperty`)、时间格式注解(如 `@JsonFormat(timezone=...)`)。

## 不默认登记(项目相关,探测确有再临时登记)

限流/防刷、多租户数据隔离、版本/向后兼容 —— **很多项目根本没有**。默认写进 spec 会**臆测出不存在的契约**(如把单租户臆测成有身份过滤)。仅当真实代码探测到才临时登记,否则不写。

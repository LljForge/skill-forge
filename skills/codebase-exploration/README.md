# codebase-exploration（代码探索 · 跨栈项目结构活地图)

帮**不熟悉某个项目**的读者快速建立结构认知:有哪些模块、各自干什么、模块间怎么调、哪里有坑、从哪读起。从源码客观测绘,产出 `docs/codebase-map.md` 一份**活地图**(落在**被探索的那个项目**里)。

> 完整运行规则见 [SKILL.md](SKILL.md),本文件只做总览与导航。

---

## 这是什么

一份**项目结构地图(活文档)**:记录慢变的架构结构(模块边界 / 依赖形状 / 约定与横切),给还不熟悉本代码库的读者 **5 分钟**建立全局认知——不是给分析师的技术报告。

- **跨栈**:Java / TS / Python / Go 等,按 manifest 探测栈、取对应配方(见 [references/stack-recipes.md](references/stack-recipes.md))。
- **不易过时**:只写抗重构的慢变结构,易变细节(行号 / 精确计数 / 穷举清单)剔除、交后续模块级深析;配漂移自检 + 一键重生。
- **双重用途**:既是团队长期的项目资产,也是 AI 编程时的架构指引。

**何时触发**(本 Skill 模型可自动触发,也可手动点名):用户想先熟悉项目全局、问「这代码库怎么分模块 / 什么结构」、或要「画个项目地图」时。

**产出 `docs/codebase-map.md` 七节**:一·技术栈与分层 · 二·模块总览 + 读者卡片 · 三·约定与横切机制 · 四·改动前必读 / 陷阱 · 五·上手顺序(✅深读 / ⚖️你定 / ⬜略读)· 六·模块依赖矩阵 · 七·新鲜度与重生。

## 边界(自包含)

本 Skill **自包含、不依赖也不读写任何其它 Skill**:产出的是**全局**结构地图——模块边界 / 依赖形状 / 约定横切。要把**某一个**模块逐行读透,是地图之外、另行处理的事;地图只在「⑤ 上手顺序」里**标出哪些模块值得后续单独深入精读**,不替读者执行那一步。

## 如何开始

在你要探索的项目里说「带我熟悉下这个项目 / 这代码库怎么分模块」即可触发;跑完产出 `docs/codebase-map.md`,从「⑤ 上手顺序」按标注逐个深入精读。

# recon-driven-dev 维护宪法

> **改本 Skill 自己**(SKILL.md / references / 模板 / 本流程)时读这份;改业务代码不读——那是本 Skill 去编排的事,不是改本 Skill。
> 这里是**治理规则、不是运行护栏**:运行时跑五阶段流水线的 AI 不需要它们,故不放 SKILL.md 脊柱、不随运行载入。**运行护栏**(报告质量尺 / 回流禁 / 评审边界 / 事实订正等)在 [`SKILL.md`](SKILL.md) 的「全局护栏」。

## 守住"薄"——防膨胀

- 🔁 **「薄」是审美、不是否决权**——吸收按"对新 Skill 好不好"逐条判,经评估保留的才留。防膨胀不靠口号:落地后**实测每条执行路径(内联 / 子 agent)单次 run 实际载入的 reference 量,设一个"单路径承载"软上限**;超了就回头评增量价值偏低的吸收项能否再降级。
- ❌ **别给定向分析加判断规则表 / 检索套路**("碰到契约就 grep 调用方"这类识别-动作对照表)——识别 AI 会,动作溶进验收单 + 坑的 why 即可,别把"薄摸底"胀成重型分析机器。

## 单一权威源(DRY · 抗漂移)

- ❌ **别把判据复述进 SKILL.md / 模板**——`review-agent.md`(设计三维度)、`implementation.md`(per-task 两阶段)、`code-reviewer.md`(整支两轴)、`requirements-design.md`(评审决策点判据 + `design.md` 必覆盖清单)各有唯一家,脊柱 / 模板**只路由不复述**(复述 = 第二权威源、会漂移)。
- 🔁 **`design.md` 不套章节骨架、也不"完全无骨架"**——改用**必覆盖清单**(四要素检查锚点,形态按复杂度伸缩)。清单的要素与判据单一权威在 `requirements-design.md`,SKILL.md 与其余只指针、**不在此列举要素**。
- ❌ **别把 provenance / 借鉴出处写进 agent 本体**(`recon-agent.md` / `review-agent.md` / `code-reviewer.md`)——可执行 prompt 只放角色 / 契约 / 判据,出处归 `CHANGELOG.md`。

## 不引持久件(设计取向)

- 🔁 **为什么否决项目级持久件**——读了长期文档反而引入漂移与冲突,故否决持久 `CONTEXT.md` / ADR 库、不发持久编号。(运行约束本身——「不引入跨会话持久件、进度账本随归档清」——在 [`SKILL.md`](SKILL.md) 全局护栏。)

## 实测打磨协议(实测整个 skill 时读)

> 本节是「拿真实开发任务实测并打磨本 skill」的**操作权威**。其余文件(`EVAL-COVERAGE.md` / `BACKLOG.md` / 监督笔记)只引用本节、不复述判据。两道防污染闸——**会话分离**、**三分诊**——保证实测纯净、且任务特例 / 模型失误不污染 skill。
> **验证只用一把尺:下面的 7 条改后验证 rubric。** 遇疑难想借诊断词汇时,可翻 writing-great-skills 的常见毛病清单(sprawl 太长 / sediment 只加不删 / no-op 说了等于没说 / duplication 重复)当参考,不把它当常驻尺。

### 何为"实测到位"

两条(地板 / 天花板判据的**唯一定义处**,收敛判定与账本均指针引用、不复述):
- **地板** — `EVAL-COVERAGE.md` 每个路径单元「触发」≥`◇`(走查)且「指令够用?」不带 `⚠`。
- **天花板**(版本封板信号,非每次实跑必达的门槛)— 被碰过的路径,连续 2 轮真实实跑不再冒出新的 skill 缺陷类改进点。

### 总览:两会话分离

```
实测会话(对象层·纯净)            打磨会话(元层·专注)
拿任意真实任务跑 skill 全程   →   读改进点清单 → 三分诊 → 处置 → 验证 → 沉淀
每⏸自评 + 跑完派复盘 agent        仅 skill 缺陷入修复;回填覆盖账本
产出:监督笔记 + 改进点清单
        └──────── 交接物:改进点清单 ────────┘
```

**会话分离**:实测会话只跑 + 记笔记,**不在此会话改 skill**;改 skill 在另起的打磨会话做——元层动作不污染实测纯净度。

### 实测会话:监督协议

**现场层(每 ⏸,exception-only)**:主 agent 在 skill 既有的每个 `⏸` 暂停点,**只在卡住 / 指令含糊 / 降级没走通时**写一行进监督笔记(带现场证据:哪句 skill 指令导致什么);顺畅就留空。零新增停点;漏记的盲区由复盘 agent 兜底,不靠主 agent 逐格自律填满。

**复盘层(整任务跑完)**:派一个**上下文隔离的复盘 agent**,输入「监督笔记 + 实走路径单元 + 全部产物」。
- 职责:专猎主 agent **照 skill 做了、却被指令误导而不自知**的盲区(自评的天然盲点)。
- 边界:只评 **skill 指令本身**在这次实跑暴露的问题,**不评业务代码质量**(那是 skill ⑤ 自己的职责)。
- 产出:补充改进点,合并进改进点清单、标 `▲复盘补`。
- 派隔离 agent 给对抗视角是本 skill 既有派发范式(①③⑤ 全用它);无隔离能力时降级主会话自跑。

**监督笔记模板**(落 `docs/recon-driven-dev-eval/<date>-<task-slug>/supervision.md`,归档即弃):

```
# 实测监督笔记 · <date> · <task-slug>
> 实跑 recon-driven-dev 全程的现场记录。元层维护资产,归档即弃。

## 走过的路径单元
（列 EVAL-COVERAGE.md 的格号）

## 逐 ⏸ 体验自评
- <阶段><⏸>  顺/卡/含糊/降级 + 现场证据  [→ IP-NN]
```

**改进点清单模板**(交接物,落同目录 `improvements.md`):

```
# 改进点清单 · <date> · <task-slug>   (交接物 → 打磨会话)
IP-NN [▲复盘补]? | 来源 <阶段/⏸/路径单元号> | 现象:<skill 指令怎么让 AI 卡/错/含糊> | 初判 <skill缺陷?/任务特例?/模型失误?> | 影响面 <每次X/某分支> | 严重 <高/中/低>
```
「初判」只是实测会话的快速归类;最终定诊在打磨会话。

### 打磨会话:分诊 → 处置 → 验证 → 沉淀

**0. 先对账 BACKLOG**(分诊前的一步)——扫 `BACKLOG.md` 既有搁置项的「待触发条件」:本次改进点 / 实跑样本若命中某条触发条件,把那条**一并拉进本会话**(连同新点一起走下面的三分诊 → 处置:攒够样本 / 想清权衡就出列定稿或正式关闭,仍不足则更新其待触发条件留着)。这一步是搁置项**唯一的出列入口**——没人主动核对,它的触发条件就烂在清单里。

**1. 三分诊**(防污染 skill 的核心闸,可派隔离对抗 agent 复核防主观)——对每条改进点定诊:
- **skill 缺陷** — skill 指令本身的问题(含糊 / 缺动作 / 降级没说清 / 停点冗余 / 判据漂移 / 跨阶段衔接断)。**仅此类入修复流程。**
- **任务特例** — 本项目 / 环境特殊导致,非 skill 通用问题。→ 可回写被开发项目的文档,**不改 skill**。
- **模型自身失误** — skill 指令没错,是 AI 没照做 / 能力波动。→ 至多提示某处措辞可加固,本质不算 skill 缺陷。

依据本宪法「通用 vs 特定」取向:复用资产只放通用约束,项目 / 环境特有事实不污染 skill。

**2. 处置门槛**(确认为 skill 缺陷后):
- **当场改** — 局部 · 低风险 · 单样本足以确认(某句指令含糊、某 ⏸ 模板缺一句):改 SKILL.md / reference + 验证 + 立 `CHANGELOG.md`。改到 ②③④⑤ 这类有外部范式渊源的能力时,顺手翻一眼 CHANGELOG 记的借鉴出处(回 superpowers / mattpocock 原版对照),看是不是当初有意砍的、别把砍掉的胀回来。
- **进 BACKLOG 搁置** — 涉护栏权衡 / 动骨架 / 单样本不足以定方向 / 疑似需多实跑样本佐证:按 `BACKLOG.md` 模板记 problem + 候选修法 A/B/C + 待触发条件。

**3. 改后验证 rubric**(改了 skill 后逐条过):

A. 结构契约没破(改对了没)
- **自包含性** — 没引入外部 skill 调用 / 脚本 / 跨会话持久件依赖。【唯一硬约束】
- **单一权威源** — 没制造第二处判据复述;若动了某判据的家,按名引用它的地方(模板维度名、产物落点 / 命名空间)同步改、没漏。
- **分层归位** — 运行约束落脊柱、治理规则落本文件、判据落 reference,没放错层。
- **派发-降级对称** — 改了派发侧契约,主会话降级侧同步;反之亦然。
- **阶段衔接·不回流** — 线性没破、把关没堆下游;**跨阶段接缝没断**:信号产出义务在没在(② 给评审决策点、② 给 ⑤ 的测试缝隙约定)· 下游真去读没 · 按名耦合的名改了没漏 · 可选分叉处判据够不够 AI 自己下刀。

B. 改动本身够格(改好了没)
- **可执行性** — 改的指令是 AI 真能照做的动作工单,不是陈述体 / 职位说明书,不依赖它没有的命令或信号。
- **单路径承载** — 改动没让某条执行路径(内联 / 子 agent)单次 run 载入的 reference 量超软上限。【防膨胀】

**怎么跑验证(强度随改动量分档)**:① 都要做**消费者映射**——改了谁 → 谁按名 / 按序消费它 → 只重验那条链,不读该产出的环节不必重跑;② **当场改**(局部低风险)主会话自核「自包含 + 单一权威」两条即可;**大改 / 动骨架 / 进 BACKLOG 的**才派**隔离对抗 agent** 逐条过全 7 条。**诚实定性**:`CHANGELOG.md` 注明本次是能力提升还是工程收益。

**4. 回填 + 沉淀**:
- 回填 `EVAL-COVERAGE.md`:本次走过的格 → 更新「触发」+「指令够用?」+ 关联改进点号 + 最近实跑日期 + 进度小结。
- 已采纳改进沉 `CHANGELOG.md`(what + why);搁置项沉 `BACKLOG.md`。

### 收敛判定

地板 + 天花板(判据见上「何为实测到位」,此处不复述)皆达 → 宣布「当前版本实测到位」,转入低频维护;skill 后续大改后,受影响路径单元的「触发」重置为 `·未触发`、按本协议重测。

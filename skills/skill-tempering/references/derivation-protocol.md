# 派生工序：现场长出"额外适用纪律"的权威定义

> 本文件是 `skill-tempering` **派生工序**的唯一权威定义处——七核（`principles.md` 的 `#1`–`#7`）之外、因被测 skill 形态不同而**额外**该套上的纪律，不靠枚举类型查表，而靠这一道现场判断工序长出。`audit-lens.md`（路线 A）/ `evidence-lens.md`（路线 B）/ `SKILL.md` 引用本工序、**不复述**（守 #5：单一权威源）。
>
> **为什么不是清单 / 分类器**：枚举「skill 类型」或「判别轴」都是开放集——每来一个新型 skill 就冒新维度，可机械兜住任意 skill 的固定脚手架并不存在（见 `../theory-foundation.md`）。故底座是「小而硬的通用核（七核）＋ 一道派生工序（本文件）」，**刻意不做成机械分类器**。本工序自身正是七核的活体应用：它用 #4（机械的归工具、判断的归模型）把"嗅证据"机械化、把"派生纪律"留给现场判断；用 #1（自评不可信）拒绝靠内省自封覆盖；用 #5（守薄）声明盲区而不假装穷尽——它吃自己的狗粮。
>
> **怎么读**：七核先过完（按 `principles.md`）之后，再走本工序六步。本工序产出的每条派生纪律，也带一个`强制力档`（取值定义见 `principles.md`「强制力档取值」节，本文件不复述）。

---

## 六步总览

```
① 归型(嗅 + 失败形态) → ② 派生条件纪律(查种子库 + 现场长) → ③ 完整性 critic(种子没问到的命脉)
  → ④ 证据来源二分(路线 A 静态推定 / 路线 B 用户实证) → ⑤ 结晶治理(pending+留痕+复核) → ⑥ 盲区声明(部分符合)
```

**①②③④⑥ 两路线每次审计都走**，差别只在 ④ 的**证据来源**（A 从文本嗅、B 吃用户带来的实证）；**⑤ 结晶治理属维护层**——仅当 ③ 当场派生出新维度、要沉回种子库前才触发，不进每次审计的产出流程（见 ⑤）。六步之外另有一道**三分诊归因闸**（见文末独立节）——它不产派生纪律，而是给"每条判出的疑似问题"归因，A/B 各按自己的证据语境用它、每次审计都走。

---

## ① 归型 = 嗅 + 失败形态（不问"你是哪类"）

**不问被测 skill"你属于哪一类"**——类型是开放集，问了也长不出可靠纪律。改为两路读**可观测事实**：

**A. 嗅可观测证据**（机械可读的事实，归 #4 的"确定性归工具"那一侧——能机械判定的别让模型贴标签）。借工业先例 `domains` 嗅探思路（点名见 `../theory-foundation.md`）。逐条读事实、**不贴类型标签**：

- 它**产持久产物**吗（产出落地为可长期留存、会被后续依赖的东西）？
- 它产**全称 / 穷尽断言**吗（声称"全部""所有""已覆盖完"这类对全集的承诺）？
- 它落**不可逆副作用**吗（执行后无法干净回退、或会改动外部世界状态）？
- 它走**多路径 / 多文件**吗（分支多、或被测 skill 自身被拆成多份载体）？
- 它的**价值在产物还是在过程**（终点是交付一个物，还是引导一段判断 / 交互）？

> 嗅的是**事实**不是**身份**：读"它产不产全称断言"，而非判"它是不是验证类 skill"。同一个 skill 可同时命中多项事实——这正是开放集下"skill 是异质复合体"的实情，比单一类型标签更接近真相。

**B. 分叉轴 = 失败形态，而非 skill 类型**。借先例「Match the Form to the Failure」（点名见 `../theory-foundation.md`）：**看缺这个 skill 时 agent 会怎么失败**，按失败形态选指导形式。失败形态从何得知随路线而异——路线 A 从文本静态推断"最可能怎么坏"，路线 B 直接读用户实证里"实际怎么坏了"（见 ④ 证据来源二分）。失败形态是一个**小而封闭的集**（与"类型开放集"相反，这是本工序敢闭合的地方）：

| 失败形态 | 含义 | 对应该补的指导形式 |
|---|---|---|
| **skips-rule** | 该守的规矩被跳过 | 把规矩写成不可跳的硬闸 / 触发条件 |
| **wrong-shape** | 产物形状错（结构 / 格式跑偏） | 给目标形状的样板 / 模板 |
| **omits-element** | 漏掉必含要素 | 给可核对的全集清单（接 #1 的完整性验证形态） |
| **condition-dependent** | 只在某些前置下才崩 | 显式覆盖那些前置的失败 / 缺失分支（接 #3） |

> A 嗅"它是什么事实"，B 嗅"它不在场时怎么坏"。两路都读证据、都不靠类型标签——这是软杀"牛头不对马嘴"误配的第一道刀。

## ② 派生条件纪律（种子库 · 表）

把 ① 嗅到的事实 / 失败形态，喂进下面这张**种子库**：命中`触发`就挂上对应`纪律`，按`强制力档`定其处置力度。每条派生纪律都从七核加嗅到的证据推出（"它从哪条核长出"列已标），**不是七核之外另起炉灶**。

| 纪律 | 触发（嗅到什么） | 强制力档 | 从哪条核长出 |
|---|---|---|---|
| **完整性验证形态**（核全集、不抽样自夸） | 产**全称 / 穷尽断言** | `soft` | #1 承诺形态决定验证形态 |
| **HITL 人控闸**（高代价动作须人放行） | **高代价 / 不可逆**决策 | `hard` | #1（无外部真值交人）＋ #3（不可逆即"失败不可承受"） |
| **慢变 · altitude ＋ 可重生 · 新鲜度**（写抗重构的慢变层、留可重新生成的活产物） | 产**持久产物** | `soft` | #5 守薄（防沉积）＋ #2（产物有"何时算过期"判据） |
| **分层披露**（脊柱 / references / 指针，别全压进一处） | **多路径 / 多文件** | `soft` | #5 单一权威源 ＋ #6（脊柱承载激活触发） |
| **副作用族**：凭证敏感 / blast-radius / 幂等重入 | 落**不可逆副作用** | `hard`（凭证 / blast-radius）· `soft`（幂等重入） | #3 失败 / 缺失路径 ＋ #1 |
| **输入族**：源受控性（输入可信吗 / 谁能投毒） | 依赖**外部输入** | `soft` | #3（输入异常分支）＋ #4 |
| **判据族**：时变（判据会过期吗）/ 效度（判据真测到了想测的吗） | 价值取决于**判据 / 断言**的对错 | `soft` | #2 完成判据 ＋ #1 |
| **边缘适用者一律降 `advisory`** | 嗅到某纪律对这个 skill 只**沾边、非命脉** | `advisory` | #4（边缘＝判断空间，留模型现场判，不硬编死规则）|
| **……（缺了当场派生，走 ③）** | 嗅到上表没有的命脉事实 | 现场定，默认从严起、待 ⑤ 复核 | 视所长出的核而定 |

> **边缘适用 → advisory，不硬套**：一条纪律只跟被测 skill 沾边、不是它的命脉时，**打 `advisory`（建议）而非强制**。这是软杀"牛头不对马嘴"的第二道刀——宁可建议、不可把不相干的纪律硬扣成 hard / soft 去逼改。判"是命脉还是沾边"本身是判断活，归模型现场判（#4），不预编成死表。

> **种子库非穷举、缺了当场派生**：上表是**种子**不是全集——skill 形态是开放集，任何枚举都是跑步机。嗅到表里没有的命脉事实，**不许硬塞进最近的一格凑数，要走 ③ 当场派生新维度**，再经 ⑤ 治理决定是否沉回种子库。**缺这一行，本工序就退化成它刻意要避开的机械分类器。**

## ③ 完整性 critic（我们独有、linter 没有的一环）

种子库再全也是过去经验的快照，挡不住没见过的新型 skill。故在 ② 之后**显式追问一句**：

> **"这个 skill 有什么命脉，是我上面的种子一条都没问到的？"**

- 这是本工序相对工业 linter 的**独有增量**——linter 只能跑既有规则集，问不出"我的规则集缺了什么"。
- 若 critic 揪出种子没覆盖的命脉，**当场派生一条新维度纪律**（拟名 + 触发 + 暂定强制力档），挂上本次审计，并交给 ⑤ 治理。
- critic 的产出**不可只靠内省自封**（#1：自评不可信）——它给的是"疑似命脉"，是否真灵要么靠 ④ 路线 B 的用户实证印证，要么（路线 A、或 B 里实证没撞到）至少在 ⑥ 盲区声明里诚实标为"待验证"。
- **追问设死上限（守 #2 完成判据）**：critic 追问不无限进行——同一 skill 建议 2–3 轮（具体轮数归模型按 skill 复杂度现场判，#4），到顶仍揪不出新命脉即停、转 ⑥ 盲区声明,不为求全而空转。**自审场景**（被审对象＝审计者自己，如拿本 skill 审本 skill）同此上限:三分诊"审计者误读 / 归因存疑"一类的自我质疑到顶不决即**退盲区、交人复核**,不陷入无限自我怀疑递归。

---

## ④ 证据来源二分（这条纪律对这个 skill 到底灵不灵，凭什么信）

③ 派生出的判断、② 命中的纪律，**别只靠 critic 内省定生死**（#1：被测对象对自身断言无可靠内部信号，本工序对自己派生的纪律同样不可自封）。每条纪律"对这个 skill 到底灵不灵"要有**证据**才采信——证据分两种来源，对应两条路线：

- **路线 A（静态推定）**：只读被测 skill 的文本载体、不实跑，靠 ①–③ 的嗅探 + 七核问句 + 完整性 critic 推断。此路证据**一律是静态推定**——凡未经实证的派生纪律，在 ⑥ 盲区声明里标为"静态推定、未实证"，绝不冒充已验证（#1）。
- **路线 B（用户实证）**：用户**已经在真实使用中跑完**这个 skill，把攒下的实证（卡点记录 / 失败案例 / 运行 trace / 使用反馈）作为输入带进来。此路证据是**真实运行留痕**——它能印证或推翻某条纪律：某条实证落在 ① 嗅的哪个失败形态上，就印证对应纪律确实在补那个洞；实证里从没撞到的纪律仍是静态推定，照 A 路标盲区。

> **skill-tempering 自己绝不实跑**（守 #7 自包含 + 甩掉评测机床）：证据要么来自 A 路的静态推断、要么来自 B 路用户**已跑完**带来的实证。本工序**不自造 baseline 双跑、不织覆盖账本、不产生实证**——产生实证是用户在真实使用（或另用 skill-creator 等评测工具）里的事，本工序只**消费**它。凡本步冒出"让 skill-tempering 去跑一遍对照"的念头，即是被砍掉的机床复辟、违 #5，打回。

## ⑤ 结晶治理（派生维度沉回种子库前的闸）

③ 当场派生的新维度，**沉回 ② 种子库之前必须过治理闸**——否则种子库会无人负责地膨胀，正是 #5 守薄要对抗的"蔓延"。借工业先例（新规则先挂 pending、引入仲裁角色；点名见 `../theory-foundation.md`）。三件**缺一不可**：

1. **`pending`（默认挂起，不自动生效）**：新派生维度先以 `pending` 态入库，**只对当次审计生效、不自动成为全局默认纪律**。下一份 skill 不自动继承它，除非复核放行。
2. **留痕**：记下它**何时、因审哪个 skill、由哪条 critic 问句、长出什么触发 / 暂定强制力档**而来——可追溯到源头，便于复核与回撤。
3. **复核**：由一个**裁判角色**（非当场派生它的同一判断者，守 #1 产出 ≠ 裁判）决定 `pending → 转正 / 维持 pending / 撤销`。单人维护场景下，至少做**异步 fresh-eyes 自审 + 留痕**（此点的落地强度仍是设计开放问题，见设计稿）。

> **不自动回灌**：派生维度**绝不**绕过上面三件直接写回种子库。自动回灌＝种子库无人负责地膨胀＝违背 #5 守薄。这是本方案唯一没有现成成熟先例、因而最该上治理闸的部件。

## ⑥ 盲区声明（结构化部分符合声明）

工序走完，**永不宣称"已覆盖所有 skill 类型 / 所有适用纪律"**（那等于假装把开放集穷尽了，#1 + #5 都禁止）。改为随产出附一份**结构化部分符合声明**（借无障碍标准的"部分符合"先例，点名见 `../theory-foundation.md`），逐行列：

| 未覆盖项 | 理由 | 责任人 |
|---|---|---|
| （这次没审到 / 没把握的命脉维度） | （为何没覆盖：种子没问到 / 静态推定未实证 / 用户实证没撞到 / 超出本次范围） | （谁来跟进） |

- 诚实声明"我没覆盖什么"，比假装全覆盖更可信，也给下一次审计与 ⑤ 结晶留下接口。
- 所有"静态推定、未实证"的派生纪律（见 ④ 路线 A，及路线 B 里实证没撞到的），都应在此显式记一行。

## 三分诊归因闸（一道闸 · 双语境 · 不占编号六步）

> **它不产派生纪律，故不在①–⑥生产线里。** 它是一道**归因闸**：每一条"判出的疑似问题"（七核逐条判 + 派生纪律命中 + critic 揪出的命脉）**进改进清单之前**，都先过它归一次因——错在谁？`audit-lens.md`（路线 A）/ `evidence-lens.md`（路线 B）**引用本闸、注明各自证据语境、不复述**（守 #5）。

**为什么要它**：#1（自评不可信）不只管被测 skill，也管**审计者自己**——审计者判出的每条问题，同样不能默认为真值，得先自问"这是 skill 的错、还是它有意为之、还是我（跑者 / 审计者）自己的问题"。不归因就进清单，会把假阳性当缺陷报，逼出不必要的改动（正是 `audit-lens.md` 反"过度报缺口"要拦的）。

**一道闸、双语境**——三类的骨架同构，具体含义随证据来源变：

| 三类（归因） | 路线 A 语境（静态证据） | 路线 B 语境（用户实证） | 出口 |
|---|---|---|---|
| **真·缺陷** | skill 文本确有机制层违规 | 实证暴露的问题确由 skill 缺陷所致 | → 进改进清单（按强制力档分级） |
| **设计取舍 / 任务特例** | 审计者把 skill **有意的负向边界**误读成"缺失" | 这次是**任务特殊**、非 skill 通病（换任务不复现） | → 不报 / 降 advisory 注明"非缺陷" |
| **归因存疑** | **审计者误读**：看走眼 / 幻觉 / 过度解读 skill 意图 | **模型失误**：这次是模型自己搞砸、不怪 skill | → 退 ⑥ 盲区声明"待确认"，不报 |

- **凭什么分得开**：A 只有文本、分诊靠推断（故"归因存疑"多、多退盲区）；B 有真实 trace，能真区分"skill 缺陷 vs 任务特例 vs 模型失误"——**这正是三分诊本为动态证据而设的本意**，在 B 路满血。
- **三个出口全是已有出口**（改进清单 / 降级丢弃 / ⑥ 盲区声明）——本闸**零新机床、零外部依赖**，只是把"报不报"细化成"按三类归档到已有出口"。

---

> 自洽性回望：本工序处理"skill 形态是开放集"这件难事的办法，正是七核自身——#4（核 / 嗅机械化、派生 / 命脉判断留模型）、#1（不自封覆盖、消费实证而不自跑、审计者自己也过三分诊、交人复核）、#5（声明盲区、治理回灌、不假装穷尽、拒机床复辟）。它优化不了自己即是坏的，故它必须吃自己的狗粮。

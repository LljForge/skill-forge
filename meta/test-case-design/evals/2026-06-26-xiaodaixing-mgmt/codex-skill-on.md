- 待测事实集（本次用例的唯一来源，`来源` 字段仅回指以下编号）：
  - 验收标准 #1：新表 `tjd_base_xiaodaixing_project` 存在，含 36 个业务列、批次列以及审计、租户、逻辑删除、版本列。
  - 验收标准 #2：biz_web dev 环境“业务办理”下可见“小贷星管理”，访问后为查询页。
  - 验收标准 #3：上传 `小贷星项目清单（20260531）.xlsx` 后，66 行全部入库且脏值保真；同批 `data_date=20260531`、`batch_no` 相同。
  - 验收标准 #4：列表支持分页，并能按客户名、项目号、所属机构、是否逾期、项目评级、数据日期筛选。
  - 验收标准 #5：交付 prod 菜单接入说明，至少明确父节点“业务办理”、路由 `/slp/xiaodaixing` 和菜单编码等配置要素。
  - 验收标准 #6：后端 base-service 可离线编译通过，且不新增 Maven 依赖。
  - 业务规则 §1：dev 菜单由本地侧栏模拟数据承载；prod 菜单权威在仓外 `qfc-base-auth`，本工作区只交付接入说明、不直接配置 prod 菜单。
  - 业务规则 §2：表结构尽量与 xlsx 的 36 列 1:1 保真，不清洗；两个“核销金额”分别存储，空表头列使用占位字段，脏值按字符串原样保留，零丢失。
  - 业务规则 §3：每次导入只追加、不按项目号去重；一次上传形成一个批次，以“数据日期”为业务批次键，首个非空值向整批补齐，并为每次导入生成一个 `batch_no` 和 `import_time`。
  - 业务规则 §4：查询采用分页，筛选项为客户名、项目号、所属机构、是否逾期、项目评级、数据日期。
  - 业务规则 §5：后端落在 base-service、表名使用 `tjd_` 前缀并复用现有 EasyExcel 4.0.3；前端复用既有 QmTable、QmForm、QmUploadFile 范式。
  - 业务规则 §6：功能范围仅含导入与查询，不提供单条新增、编辑、删除和导出，也不在本工作区直接配置 prod 菜单。
  - 异常场景 #1：上传空文件时提示“文件不能为空”，且不入库。
  - 异常场景 #2：上传扩展名不是 `.xls` 或 `.xlsx` 的文件时提示“格式不支持”，且不入库。
  - 异常场景 #3：工作簿含重复列名、空表头列或“是否逾期”等日期拼接脏值时，导入不报错、不丢行并原样保真。
  - 异常场景 #4：“数据日期”列全为空时，以本次上传时间作为整批 `data_date` 的兜底值。
  - 接口契约 #1：导入端点为 `POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/importExcel`，接收一个 multipart 文件并返回 `Result`；`.xls` 与 `.xlsx` 是受支持格式。
  - 接口契约 #2：导入按列序 0–35 映射全 String Excel DTO，再转换并追加保存；36 列及其字段类型以设计映射表为准，金额列转换应对空值安全。
  - 接口契约 #3：导入时同批各行取得同一 `batch_no` 和 `import_time`，请求期用户标识写入 `create_user`；首个非空 `data_date` 补齐整批，全空则使用上传时间。
  - 接口契约 #4：查询端点为 `POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/page`，接收 `SearchVO` 并返回分页结果，只查询 `is_deleted=0` 的记录。
  - 接口契约 #5：查询契约包含客户名、项目号、所属机构、是否逾期、项目评级、数据日期六个业务筛选项及分页输入。
  - 接口契约 #6：前端路由为 `/slp/xiaodaixing`；查询页包含筛选表单、分页表格与上传能力，上传请求携带 `getConfigHeaders()` 所提供的配置头。
  - 接口契约 #7：新表主键为 `id varchar(64)`；批次列为 `batch_no varchar(64)`、`import_time datetime`；并含 `tenant_id`、`create_user`、`create_time`、`last_modify_user`、`last_modified`、`is_deleted tinyint default 0`、`version bigint`。

# 测试用例规范 — 小贷星管理（xiaodaixing-mgmt）

## 概览

- **输入来源**：
  - 需求 `docs/recon-driven-dev/_archived/2026-06-26-xiaodaixing-mgmt/requirements.md`
  - 设计 `docs/recon-driven-dev/_archived/2026-06-26-xiaodaixing-mgmt/design.md`
- **任务面**：后端接口（导入 + 分页查询）为主；菜单挂载 / prod 接入文档 / 前端路由 / 构建约束属**非后端接口**，用维度表点名扫的只有后端半（见覆盖矩阵备注）。

### 待测事实集（本次用例的唯一来源，`来源` 字段回指此处编号）

**验收标准**
- **验收 AC1**：新表 `tjd_base_xiaodaixing_project` 建好，含 36 业务列 + 批次列 + 审计/租户/逻辑删除/版本列。
- **验收 AC2**：biz_web dev 环境「业务办理」下出现「小贷星管理」菜单，点进去是查询页。
- **验收 AC3**：上传 `小贷星项目清单（20260531）.xlsx` → 66 行全部入库（含脏值保真），同批 `数据日期=20260531`、同一 `batch_no`。
- **验收 AC4**：列表分页可用，常用筛选项可查。
- **验收 AC5**：产出 prod 菜单接入操作说明文档。
- **验收 AC6**：后端可离线编译通过，无新增 Maven 依赖。

**业务规则**
- **规则 R1**：菜单挂载——dev 改前端 mock；prod 菜单权威在仓外 `qfc-base-auth`，本次仅出接入说明，不在本工作区配 prod 菜单。
- **规则 R2**：表结构 1:1 保真——36 列不清洗；重复列名「核销金额」加后缀区分；空表头列起占位名；`是否逾期` 等脏值列原样入 `varchar`；零丢失。
- **规则 R3**：导入 = 只追加不去重 + 批次概念——每次导入只追加、不按项目号去重；以 Excel「数据日期」列作批次键（仅首行有值、fill-down 到整批）；另加 `batch_no`(每次一个 UUID) + `import_time` 兜底。
- **规则 R4**：查询 = 分页 + 常用筛选（客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期(批次) 等）。
- **规则 R5**：复用既有依赖与范式（EasyExcel 4.0.3、`tjd_` 前缀、前端 projects 页范式），零新引依赖。

**异常场景**
- **异常 EX1**：上传空文件 / 非 Excel（非 .xls/.xlsx）→ 期望提示「文件不能为空 / 格式不支持」，不入库。
- **异常 EX2**：Excel 含脏数据（重复列名、空表头列、`是否逾期` 日期拼接脏值）→ 原样保真入库、不报错、不丢行。
- **异常 EX3**（风险①）：月度重复导入同一份清单 → 产生重复行（R3 只追加的预期代价，用 `batch_no` 区分批次缓解）。
- **异常 EX4**（风险②）：「数据日期」若某月 Excel 首行也为空 → 该批次业务日期缺失，需导入时容错（取上传时间兜底）。

**接口契约**
- **契约 IC1**：`POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/importExcel`，入参 Excel 文件，出参 Result。
- **契约 IC2**：`POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/page`，入参查询条件（SearchVO），出参分页结果。
- **契约 IC3**：前端路由 `/slp/xiaodaixing`。
- **契约 IC4**：导入内部逻辑——EasyExcel 全 String 读入（`@ExcelProperty(index=0..35)`）→ 转类型（null 安全）+ `数据日期` fill-down + 填 `batch_no`/`import_time` + `create_user=AppContext.getUserId()` → `saveBatch`（只追加、不去重）。
- **契约 IC5**：查询——`BaseTableServiceImpl` 分页，**手写 `is_deleted=0`** 条件（无 `@TableLogic` 自动过滤）。
- **契约 IC6**：36 列 index→列名→类型映射表（金额清晰列 `decimal(24,6)`；脏值/日期序列号/比例列 `varchar` 保真）。含重复列 idx16→`write_off_amount` / idx30→`write_off_amount_2`、空表头列 idx34→`guarantee_way_detail`、脏值列 idx18→`is_overdue`、比例列 idx24→`reserve_ratio`。

## 1. 功能测试 (TC-F)

#### TC-F-001 · 标准月度清单导入落库
- **维度**：—（功能主流程）
- **来源**：验收 AC3 / 规则 R3 / 契约 IC1 / 契约 IC4
- **优先级**：高
- **GIVEN** 一份符合清单结构的月度 Excel（66 行数据、首行「数据日期」为 20260531）
- **WHEN** 通过导入端点提交该文件
- **THEN** 66 行全部落入台账，无丢行
- **THEN** 该批全部行 `data_date` 一致为 20260531、共享同一 `batch_no`、均带 `import_time`

#### TC-F-002 · 导入行携带操作人身份
- **维度**：—（功能主流程）
- **来源**：契约 IC4（`create_user=AppContext.getUserId()`）
- **优先级**：中
- **GIVEN** 一次由已登录运营人员发起的导入
- **WHEN** 该批数据落库
- **THEN** 落库行的创建人字段为当次操作人的用户标识（非空、非框架缺省的空值）

#### TC-F-003 · 数据日期 fill-down 整批对齐
- **维度**：—（功能主流程；关联 §1 输入边界）
- **来源**：规则 R3 / 契约 IC4
- **优先级**：高
- **GIVEN** 一份 Excel，其「数据日期」仅首行有值、其余数据行该列为空
- **WHEN** 导入该文件
- **THEN** 该批所有行的 `data_date` 统一取首个非空值，不出现「首行有值、后续行为空」的批内不一致

#### TC-F-004 · 默认分页查询台账
- **维度**：—（功能主流程；关联 D5 成功信封）
- **来源**：验收 AC4 / 规则 R4 / 契约 IC2 / 契约 IC5
- **优先级**：高
- **GIVEN** 台账中已存在若干未逻辑删除的项目行
- **WHEN** 不带筛选条件发起分页查询
- **THEN** 返回分页后的台账列表，仅含 `is_deleted=0` 的行
- **THEN** ⟨预期待定 · 源未定义成功信封结构（分页字段名、总数/页码/页大小字段、缺省页大小）；见待澄清 #1⟩

#### TC-F-005 · 单一常用条件筛选命中
- **维度**：—（功能主流程；关联 §1 输入边界）
- **来源**：规则 R4 / 验收 AC4
- **优先级**：中
- **GIVEN** 台账中存在可被某一常用筛选字段（如客户名 / 项目号 / 所属机构 / 项目评级 / 数据日期批次）命中的行
- **WHEN** 指定该单一筛选条件发起查询
- **THEN** 结果仅含匹配该条件的行

#### TC-F-006 · 金额清晰列正常数值入库
- **维度**：§1 输入边界
- **来源**：契约 IC6（金额列 `decimal(24,6)`）/ 规则 R2
- **优先级**：中
- **GIVEN** 一份 Excel，其合同金额 / 放款金额等清晰金额列为合法数值
- **WHEN** 导入该文件
- **THEN** 对应 `decimal(24,6)` 列以数值形态落库，精度不被截断为整数或丢失小数位

## 2. 边界测试 (TC-E)

#### TC-E-001 · 仅表头、零数据行的 Excel
- **维度**：§1 输入边界（集合大小下界）
- **来源**：契约 IC1 / 契约 IC4（导入为集合写入，行数是其输入边界）
- **优先级**：中
- **GIVEN** 一份格式合法但不含任何数据行的 Excel
- **WHEN** 导入该文件
- **THEN** ⟨预期待定 · 源未定义零行导入语义（建一个空批次并成功 / 判为无有效数据而拒绝）；见待澄清 #4⟩

#### TC-E-002 · 数据日期首行也为空的容错兜底
- **维度**：§1 输入边界（必填缺失）
- **来源**：异常 EX4（风险②）/ 契约 IC4
- **优先级**：高
- **GIVEN** 一份 Excel，其「数据日期」列首行即为空（整列无可 fill-down 的值）
- **WHEN** 导入该文件
- **THEN** 该批 `data_date` 以上传时间兜底填充，导入不因业务日期缺失而失败，行不丢失

#### TC-E-003 · 金额清晰列含非数字脏值
- **维度**：§1 输入边界（类型错）
- **来源**：规则 R2 / 契约 IC4（转类型 null 安全）
- **优先级**：中
- **GIVEN** 一份 Excel，其某个 `decimal` 金额列的值为非数字脏字符
- **WHEN** 导入该文件
- **THEN** 该单元格转为空值落库，导入不报错、该行其余列不丢失

#### TC-E-004 · 重复列名「核销金额」双列保真
- **维度**：§1 输入边界 / 关联 D5 契约
- **来源**：规则 R2 / 契约 IC6（idx16→`write_off_amount`、idx30→`write_off_amount_2`）
- **优先级**：中
- **GIVEN** 一份 Excel，含两个同名「核销金额」列（含非数字值）
- **WHEN** 导入该文件
- **THEN** 两列分别原样落入两个后缀区分的列，互不覆盖、值保真

#### TC-E-005 · 空表头占位列保真
- **维度**：§1 输入边界
- **来源**：规则 R2 / 契约 IC6（idx34→`guarantee_way_detail`）
- **优先级**：中
- **GIVEN** 一份 Excel，其第 35 列（idx34）表头为空、单元格有值（如「信用贷款」）
- **WHEN** 导入该文件
- **THEN** 该列值原样落入占位列，不因表头缺失而丢列或丢值

#### TC-E-006 · 是否逾期脏值（日期拼接）原样入库
- **维度**：§1 输入边界
- **来源**：规则 R2 / 异常 EX2 / 契约 IC6（idx18→`is_overdue` varchar）
- **优先级**：中
- **GIVEN** 一份 Excel，其「是否逾期」列为日期拼接类脏值
- **WHEN** 导入该文件
- **THEN** 该值原样落入 `varchar` 列，不被解析/转换、不报错

#### TC-E-007 · 日期序列号列原样入库
- **维度**：§1 输入边界
- **来源**：规则 R2 / 契约 IC6（idx19/20 本金/利息逾期起始日 varchar）
- **优先级**：低
- **GIVEN** 一份 Excel，其逾期起始日列为 Excel 日期序列号
- **WHEN** 导入该文件
- **THEN** 该值原样落入 `varchar` 列，不被还原为日期、不报错

#### TC-E-008 · 准备金计提比例保真
- **维度**：§1 输入边界
- **来源**：规则 R2 / 契约 IC6（idx24→`reserve_ratio` varchar）
- **优先级**：低
- **GIVEN** 一份 Excel，其「准备金计提比例」列为比例值（如 0.6）
- **WHEN** 导入该文件
- **THEN** 该值原样落入 `varchar` 列，不被折算为百分数或数值

#### TC-E-009 · 分页参数边界取值
- **维度**：§1 输入边界（数值区间 / 分页参数）
- **来源**：规则 R4 / 契约 IC2（分页存在，参数是其输入边界）
- **优先级**：中
- **GIVEN** 台账中存在多页数据
- **WHEN** 以边界页码 / 页大小（如页码 0 或超末页、页大小 0 / 极大值 / 负值）发起查询
- **THEN** ⟨预期待定 · 源未定义分页参数取值域、缺省值与越界行为（钳制 / 报错 / 返回空页）；见待澄清 #5⟩

#### TC-E-010 · 筛选无命中
- **维度**：§1 输入边界（等价类 · 空结果）
- **来源**：规则 R4 / 契约 IC2
- **优先级**：低
- **GIVEN** 台账中不存在满足某筛选条件的行
- **WHEN** 指定该筛选条件发起查询
- **THEN** 返回空的分页结果，不报错

#### TC-E-011 · 重复导入同一份清单
- **维度**：D1 幂等/去重（幂等半边）
- **来源**：规则 R3 / 异常 EX3（风险①）
- **优先级**：高
- **GIVEN** 某份月度清单已被完整导入过一次
- **WHEN** 再次导入同一份清单
- **THEN** 第二次以追加方式再落一整批相同业务数据，不去重、不覆盖
- **THEN** 两批数据以不同 `batch_no` 区分，同一项目号在库中并存多行

## 3. 错误处理 (TC-ERR)

#### TC-ERR-001 · 上传空文件
- **维度**：§1 输入边界（必填缺失）/ 关联 D5 错误契约
- **来源**：异常 EX1
- **优先级**：中
- **GIVEN** 一次导入操作，其文件为空
- **WHEN** 提交该文件
- **THEN** 拒绝入库，返回「文件不能为空」性质的错误提示
- **THEN** ⟨预期待定 · 源未定义错误响应结构（HTTP 状态码 / 业务错误码 / 错误体字段）；见待澄清 #1⟩

#### TC-ERR-002 · 上传非 Excel 格式文件
- **维度**：§1 输入边界（格式错）/ 关联 D5 错误契约
- **来源**：异常 EX1
- **优先级**：中
- **GIVEN** 一次导入操作，其文件非 .xls/.xlsx（如 .txt / .csv）
- **WHEN** 提交该文件
- **THEN** 拒绝入库，返回「格式不支持」性质的错误提示
- **THEN** ⟨预期待定 · 源未定义错误响应结构（HTTP 状态码 / 业务错误码 / 错误体字段）；见待澄清 #1⟩

## 4. 状态迁移 (TC-ST)

> 本任务为**只追加 + 只读查询**（需求 §5 非目标明确不做单条增/改/删），业务数据无生命周期状态机；唯一与状态相关的可测契约是**逻辑删除位对查询可见性的影响**。

#### TC-ST-001 · 逻辑删除行不进入查询结果
- **维度**：—（查询过滤契约；关联 D5）
- **来源**：契约 IC5（手写 `is_deleted=0`，坑3 无自动逻辑删除）
- **优先级**：中
- **GIVEN** 台账中同时存在 `is_deleted=0` 与 `is_deleted=1` 的行
- **WHEN** 发起分页查询
- **THEN** 结果仅含 `is_deleted=0` 的行，逻辑删除行不出现

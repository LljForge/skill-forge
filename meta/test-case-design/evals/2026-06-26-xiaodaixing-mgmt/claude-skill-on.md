### 待测事实集（本次用例的唯一来源，`来源` 字段回指此处编号）

**验收标准**（归一自 requirements §4）
- 验收标准 #1：新表 `tjd_base_xiaodaixing_project` 建好，含 36 业务列 + 批次列 + 审计/租户/逻辑删除/版本列。
- 验收标准 #2：biz_web dev 环境「业务办理」下出现「小贷星管理」菜单，点进去是查询页。
- 验收标准 #3：上传 `小贷星项目清单（20260531）.xlsx` → 66 行全部入库（含脏值保真），同批 `数据日期=20260531`、同一 `batch_no`。
- 验收标准 #4：列表分页可用，常用筛选项可查。
- 验收标准 #5：产出 prod 菜单接入操作说明文档（父节点=业务办理、路由 `/slp/xiaodaixing`、编码等）。
- 验收标准 #6：后端可离线编译通过，无新增 Maven 依赖。

**业务规则**（归一自 requirements §3）
- 业务规则 R1：菜单挂载——dev 改前端 mock 即可本地见菜单；prod 菜单权威在仓外 qfc-base-auth，本次仅出接入说明。
- 业务规则 R2：表结构 1:1 保真 36 列、不清洗——重复列名「核销金额」加后缀区分（`write_off_amount` / `write_off_amount_2`）、空表头列起占位名、`是否逾期` 等脏值列原样入 varchar、零丢失。
- 业务规则 R3：导入只追加不去重 + 批次概念——以 Excel「数据日期」列作业务批次键（xlsx 仅首行有值，导入时 fill-down 到整批），另加 `batch_no`（每次导入一个 UUID）+ `import_time`。
- 业务规则 R4：查询分页 + 常用筛选，筛选项含 客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期(批次) 等。
- 业务规则 R5：复用既有依赖与范式——导入复用 EasyExcel（零新引依赖），前端复用 projects 页 QmTable/QmForm/QmUploadFile 范式。

**异常场景**（归一自 requirements §2.2 + §6 风险）
- 异常场景 #1：上传空文件 / 非 Excel（非 .xls/.xlsx）→ 提示「文件不能为空 / 格式不支持」，不入库。
- 异常场景 #2：Excel 含脏数据（重复列名、空表头列、`是否逾期` 日期拼接脏值）→ 原样保真入库、不报错、不丢行。
- 异常场景 #3（风险①）：月度重复导入同一份清单会产生重复行（R3 只追加的预期代价，用 batch_no 区分批次缓解）。
- 异常场景 #4（风险②）：「数据日期」若某月 Excel 首行也为空，则该批次业务日期缺失，导入需容错（取文件名/上传时间兜底）。

**接口契约**（归一自 design ① 架构/数据流、③ 映射表、ADR）
- 接口契约 #1：`POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/importExcel`（`MultipartFile` → `Result`），Service 解析入库（EasyExcel 全 String DTO、index 映射 → toPO → saveBatch，只追加不去重）。
- 接口契约 #2：`POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/page`（`SearchVO` → 分页），走 BaseTableServiceImpl 分页，**手写 `is_deleted=0` 条件**（无 `@TableLogic`，不靠自动逻辑删除）。
- 接口契约 #3：前端路由 `/slp/xiaodaixing` 为查询页。
- 接口契约 #4：`toPO` 纯逻辑——类型转换 null 安全 + `数据日期` fill-down + `batch_no`/`import_time`/`create_user=AppContext.getUserId()` 填充。
- 接口契约 #5：36 列映射——金额清晰列 `decimal(24,6)`、脏值/比例/日期序列号列 `varchar` 保真；重复列名加后缀、空表头列占位名。

## 3. 错误处理 (TC-ERR)

#### TC-ERR-001 · 上传空文件被拒
- **维度**：D5 错误响应契约（错误码待澄清）
- **来源**：异常场景 #1 / 接口契约 #1
- **优先级**：高
- **GIVEN** 一个空文件
- **WHEN** 通过 importExcel 端点上传
- **THEN** 返回「文件不能为空」类提示、不入库
- **THEN**（对应 HTTP 状态码 / 业务错误码事实集未定义，见「待澄清」）

#### TC-ERR-002 · 上传非 Excel 格式被拒
- **维度**：D5 错误响应契约（错误码待澄清）
- **来源**：异常场景 #1 / 接口契约 #1
- **优先级**：高
- **GIVEN** 一个扩展名非 .xls/.xlsx 的文件（如 .csv / .txt / .pdf）
- **WHEN** 通过 importExcel 端点上传
- **THEN** 返回「格式不支持」类提示、不入库
- **THEN**（对应 HTTP 状态码 / 业务错误码事实集未定义，见「待澄清」）

#### TC-ERR-003 · 重复导入产生重复行（只追加不去重）
- **维度**：D1 幂等/去重（幂等半边——事实集明确「不去重、只追加」）
- **来源**：异常场景 #3（风险①）/ 业务规则 R3
- **优先级**：高
- **GIVEN** 某份清单此前已导入过一次
- **WHEN** 再次导入完全相同的清单
- **THEN** 第二次导入不去重、不覆盖，按追加方式再落入等量新行
- **THEN** 两次导入的行分属不同 `batch_no`，可据此区分批次

#### TC-ERR-004 · 含脏数据整批不中断、零丢行
- **维度**：业务规则 R2（保真容错）/ 异常场景 #2
- **来源**：异常场景 #2 / 业务规则 R2
- **优先级**：高
- **GIVEN** 一份混有多种脏值（脏表头、脏格值、日期序列号）的清单
- **WHEN** 导入入库
- **THEN** 全部行成批落库、无一行因脏值被中断或丢弃，导入整体不报错

## 4. 状态迁移 (TC-ST)

**本任务无业务状态机——无状态迁移用例。**

需求非目标明确「不做新增/编辑/删除单条记录」，记录入库后不流转状态；`is_deleted` 恒为 0（不提供逻辑删除入口）；「批次」是分组键而非状态，无合法/非法迁移。故本分区无用例，非遗漏。（`is_deleted=0` 的查询过滤已由 TC-F-006 覆盖，属查询契约而非状态迁移。）

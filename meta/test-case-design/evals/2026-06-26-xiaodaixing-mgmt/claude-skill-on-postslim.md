### 待测事实集（本次用例的唯一来源，`来源` 字段回指此处编号）

**验收标准**
- **验收标准 #1**：新表 `tjd_base_xiaodaixing_project` 含 36 业务列 + 批次列 + 审计/租户/逻辑删除/版本列（DDL 增量脚本）。
- **验收标准 #2**：biz_web dev 环境「业务办理」下出现「小贷星管理」菜单，进入为查询页。
- **验收标准 #3**：上传 `小贷星项目清单（20260531）.xlsx` → 66 行全部入库（含脏值保真），同批 `数据日期=20260531`、同一 `batch_no`。
- **验收标准 #4**：列表分页可用，常用筛选项可查。
- **验收标准 #5**：产出 prod 菜单接入操作说明文档（父节点=业务办理、路由 `/slp/xiaodaixing`、编码等）。
- **验收标准 #6**：后端可离线编译通过、无新增 Maven 依赖。

**业务规则**
- **业务规则 R1**：菜单挂载——dev 改前端 mock 即可见；prod 菜单权威在仓外 `qfc-base-auth`，本次只出接入说明、不直接配 prod 菜单。
- **业务规则 R2**：表结构 1:1 保真——保留 xlsx 全部 36 列不清洗；重复列名「核销金额」加后缀区分（`write_off_amount` / `write_off_amount_2`）；空表头列起占位名；`是否逾期` 等脏值列原样入 `varchar`；零丢失。
- **业务规则 R3**：导入只追加不去重 + 批次概念——每次导入只追加、不按项目号去重；以 Excel「数据日期」列（仅首行有值，导入时 fill-down 到整批）作业务批次键，另加 `batch_no`（每次导入一个 UUID）+ `import_time` 兜底。
- **业务规则 R4**：查询 = 分页 + 常用筛选，筛选项含 客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期(批次) 等。
- **业务规则 R5**：复用既有依赖与范式——EasyExcel 4.0.3（零新引依赖）、前端复用 `pages/projects` 的 QmTable/QmForm/QmUploadFile。

**异常场景**
- **异常场景 #1**：上传空文件 / 非 Excel（非 .xls/.xlsx）→ 提示「文件不能为空 / 格式不支持」，不入库。
- **异常场景 #2**：Excel 含脏数据（重复列名、空表头列、`是否逾期` 为日期拼接脏值、日期序列号、比例值）→ 原样保真入库、不报错、不丢行。
- **异常场景 #3**：「数据日期」首行也为空 → 该批业务日期缺失，导入时容错，取兜底值（上传时间）填充。
- **异常场景 #4**：月度重复导入同一份清单 → 产生重复行（只追加的预期代价），用 `batch_no` 区分批次。

**接口契约**
- **接口契约 #1**：`POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/importExcel`，入参 Excel 文件，出参 Result；全 String 读入 → toPO → 只追加入库（不去重）。
- **接口契约 #2**：`POST /api-zjq/xiaodaixing/tjdBaseXiaodaixingProject/page`，入参筛选 + 分页条件，出参分页结果；查询以手写 `is_deleted=0` 过滤（无 `@TableLogic` 自动逻辑删除过滤）。
- **接口契约 #3**：表 `tjd_base_xiaodaixing_project`——36 业务列 1:1 保真（重复列名后缀、空表头占位列、脏/日期序列号/比例列为 `varchar`、金额清晰列为 `decimal(24,6)`）+ 批次列（`batch_no`/`import_time`/`data_date`）+ 审计/租户/逻辑删除（`is_deleted tinyint default 0`）/版本列。
- **接口契约 #4**：toPO 纯转换——全 String → 类型转换（金额 decimal 且 null 安全；脏值/比例/日期序列号列 varchar 原样）+ `数据日期` fill-down 全批 + `batch_no`(UUID)/`import_time`(now)/`create_user`(`AppContext.getUserId()`) 填充。
- **接口契约 #5**：筛选项——客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期(批次) 等。

## 1. 功能测试 (TC-F)

#### TC-F-001 · 标准月度 Excel 全量入库并归批
- **维度**：§1 输入边界（合法 .xlsx 有效类代表值）
- **来源**：验收标准 #3 / 业务规则 R3 / 接口契约 #1、#4
- **优先级**：高
- **GIVEN** 一份含 66 行数据、且「数据日期」列仅首行有值为 `20260531` 的月度小贷星 Excel 清单
- **WHEN** 向导入端点提交该文件
- **THEN** 66 行全部入库、无丢行
- **THEN** 同批全部行的 `data_date` 经 fill-down 后一致为 `20260531`
- **THEN** 同批全部行共享同一个 `batch_no`，并带有 `import_time`

#### TC-F-002 · 导入后按批次查询可见（写后读到新值）
- **维度**：D4 缓存读一致性（设计直连库读写、无缓存层，写后立即可见）
- **来源**：验收标准 #3 / 业务规则 R3 / 接口契约 #2
- **优先级**：高
- **GIVEN** 一批数据日期为 `20260531` 的台账已成功导入
- **WHEN** 随即按该批次（数据日期 / batch_no）分页查询
- **THEN** 能查到该批次全部 66 行，字段值与导入内容一致，无「读到旧值」的缺失窗口

#### TC-F-003 · 无筛选分页查询返回成功信封且只含未删除记录
- **维度**：D5 错误响应契约 + 成功信封（成功信封半边）
- **来源**：业务规则 R4 / 接口契约 #2
- **优先级**：高
- **GIVEN** 表中已有多批台账数据
- **WHEN** 不带筛选条件发起分页查询
- **THEN** 返回当前页记录与分页元信息（总数、页码）
- **THEN** 结果仅包含未被逻辑删除（`is_deleted=0`）的记录

#### TC-F-004 · 常用筛选项查询命中
- **维度**：§1 输入边界（筛选字段有效类）
- **来源**：业务规则 R4 / 接口契约 #5
- **优先级**：中
- **GIVEN** 已有含不同 客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期 的台账数据
- **WHEN** 按其中一项筛选条件（如项目号）分页查询
- **THEN** 返回结果仅为满足该筛选条件的记录
- （注：筛选匹配语义——精确/模糊、大小写、多条件 AND/OR——事实集未定义，见「待澄清」，本条只断言筛选生效、不规定匹配方式）

#### TC-F-005 · 前端查询页可达并展示分页列表（前端）
- **维度**：无领域维度（前端薄查询页，纪律层适用、后端 6 维不套用）
- **来源**：验收标准 #2 / 业务规则 R1（dev 半边）
- **优先级**：中
- **GIVEN** dev 环境「业务办理」下已挂载「小贷星管理」菜单
- **WHEN** 运营人员进入该菜单对应页面
- **THEN** 页面展示小贷星台账分页列表与筛选表单

## 2. 边界测试 (TC-E)

#### TC-E-001 · 数据日期首行为空时兜底填充
- **维度**：§1 输入边界（批次键缺失的无效/缺失类容错）
- **来源**：异常场景 #3 / 业务规则 R3 / 接口契约 #4
- **优先级**：高
- **GIVEN** 一份「数据日期」列首行即为空的月度 Excel 清单
- **WHEN** 导入该文件
- **THEN** 该批 `data_date` 以兜底值（上传时间）填充整批，不因批次日期缺失而失败或丢行
- （注：兜底口径 requirements 风险②提「文件名/上传时间」两候选、design ADR-4 拍板「上传时间」，本条按「上传时间」；若最终口径不同需回改，见「待澄清」）

#### TC-E-002 · 脏数据 1:1 保真入库、零丢失
- **维度**：§1 输入边界（脏值/无效类被容纳而非拒绝）
- **来源**：异常场景 #2 / 业务规则 R2 / 接口契约 #3、#4 / 验收标准 #1
- **优先级**：高
- **GIVEN** 一份含以下脏数据的 Excel：重复列名「核销金额」两列、一个空表头列、`是否逾期` 为日期拼接脏值、逾期起始日为 Excel 日期序列号、准备金计提比例为小数（如 0.6）
- **WHEN** 导入该文件
- **THEN** 全部行入库、不报错、不丢行
- **THEN** 两个重复「核销金额」列分别落入 `write_off_amount` 与 `write_off_amount_2`
- **THEN** 空表头列落入占位列，脏值 / 日期序列号 / 比例值原样保存于对应 `varchar` 列

#### TC-E-003 · 金额列空单元格 null 安全入库
- **维度**：§1 输入边界（金额有效区间的空值边界）
- **来源**：接口契约 #4（类型转换 null 安全）/ 业务规则 R2
- **优先级**：中
- **GIVEN** 一份某金额列（如放款金额）存在空单元格的 Excel
- **WHEN** 导入该文件
- **THEN** 该金额列对应记录入库为 null，不因空值转换失败、不丢行

#### TC-E-004 · 重复导入同一份清单追加、不去重、按批次区分
- **维度**：D1 幂等/去重（幂等半边——本任务明确为「只追加、不去重」，非幂等）
- **来源**：异常场景 #4 / 业务规则 R3 / requirements 风险①
- **优先级**：高
- **GIVEN** 已导入过一份清单，生成批次 `batch_no A`
- **WHEN** 再次导入同一份清单
- **THEN** 新一批以新 `batch_no B` 追加入库，旧批次数据保留
- **THEN** 不按项目号去重、不覆盖既有行、不报冲突错误
- **THEN** 两批数据可按 `batch_no` 区分

## 3. 错误处理 (TC-ERR)

#### TC-ERR-001 · 上传空文件被拒、不入库
- **维度**：D5 错误响应契约 + §1 输入边界（必填缺失）
- **来源**：异常场景 #1 / 接口契约 #1
- **优先级**：高
- **GIVEN** 一个空文件（无内容）
- **WHEN** 向导入端点提交该文件
- **THEN** 返回「文件不能为空」类错误提示
- **THEN** 无任何数据入库
- （注：具体业务错误码 / HTTP 状态码 / 错误体结构事实集未给，见「待澄清」，本条只断言「被拒 + 提示语义 + 不入库」）

#### TC-ERR-002 · 上传非 Excel 格式被拒、不入库
- **维度**：D5 错误响应契约 + §1 输入边界（格式错）
- **来源**：异常场景 #1 / 接口契约 #1
- **优先级**：高
- **GIVEN** 一个非 `.xls`/`.xlsx` 格式的文件（如 .txt / .pdf）
- **WHEN** 向导入端点提交该文件
- **THEN** 返回「格式不支持」类错误提示
- **THEN** 无任何数据入库

## 待澄清

1. **分页参数契约**：`pageNum`/`pageSize` 字段名、缺省页大小、页大小上限、页码超出总页数时的行为——事实集只说「分页」（业务规则 R4）未给具体值，§1 输入边界无法完整推导分页字段的等价类/边界。
2. **筛选匹配语义**：客户名/项目号/所属机构等筛选项是精确匹配还是模糊匹配、是否大小写敏感、多条件之间是 AND 还是 OR——业务规则 R4 只列筛选项未定义匹配方式（影响 TC-F-004 的 THEN 精度）。
3. **D2 鉴权失败三态契约**：无凭据 / 凭据过期 / 凭据有效但无权限 分别返回什么（HTTP 状态码 / 业务错误码）；`importExcel` 与 `page` 是否需登录/特定权限——事实集有鉴权上下文线索但未定义失败契约。
4. **D3 导入事务边界**：`saveBatch`（66 行）执行期部分失败（DB 约束冲突 / 连接中断 / 某行转换异常）时，是整批回滚还是留部分脏数据、有无补偿/重试——事实集未明确写事务边界。
5. **D5 错误响应/成功信封细节**：空文件、非法格式返回的具体业务错误码与 HTTP 状态码、错误体结构；分页成功信封的具体字段名与缺省值——事实集只给错误提示文案（异常场景 #1），未给码/结构。
6. **空数据 Excel（有表头、无数据行）导入行为**：入 0 行成功 还是 报错——异常场景 #1 只覆盖「空文件（0 字节）」，未覆盖「合法 Excel 但无数据行」这一等价类。
7. **数据日期首行空兜底口径**：requirements 风险②列「取文件名/上传时间兜底」两候选，design ADR-4 拍板「上传时间」，二者措辞不一——若最终口径取文件名，TC-E-001 的 THEN 需回改。

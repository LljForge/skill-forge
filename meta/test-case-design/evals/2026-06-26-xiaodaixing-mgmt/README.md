# 对照实验 · 样本 #2 · 小贷星管理 xiaodaixing-mgmt（2026-06-26）

test-case-design 的**第二个 2×2 样本**，专为「维度表机制到底有没有差异化价值」这个悬而未决的问题挑的**鉴别性任务**。样本 #1（edoc）见 [`../2026-07-17-edoc-ownership/`](../2026-07-17-edoc-ownership/)，其结论：维度表在后端显然维度/后端隐性D3/前端全部三处都没挣到差异化价值——但那是因为 edoc 的风险面在 design 里全写明了，维度表没机会补位。

## 为什么这个任务能鉴别（edoc 不能）

被测任务：biz_web「小贷星管理」——Excel 台账导入 + 分页查询，后端为主（base-service 新表 + `importExcel`/`page` 两端点），前端极薄（一个查询页）。原始产物由 recon-driven-dev 生成。

**关键：领域逻辑写透、横切契约留白。** 36 列脏值映射/批次 fill-down 写得极细，但维度表覆盖的横切维度是欠文档的：

| 维度 | 文档状态 | 维度表的机会 |
|---|---|---|
| D3 事务边界/回滚 | **没写** importExcel `saveBatch` 部分失败回不回滚 | 裸模型很可能漏 → D3 点名正中 |
| D4 缓存读一致性 | **全篇零字提缓存** | 触发**悬案 1**（事实集沉默时问不问缓存），供裁决数据点 |
| D5 错误码契约 | 只有提示文案，无错误码/错误体/分页信封 | 裸模型大概率只写「返回失败」→ D5 逼出错误码 |
| D2 鉴权/多租户 | `tenant_id` 列在、但端点鉴权/查询隔离没写 | §3 边界情形，看两边怎么处理 |

**额外结构诱饵**：design §④ 测试缝隙只点 `toPOList` 映射单测一条、还给失败测试处方——会把裸模型注意力往映射逻辑带、偏离横切维度。维度表点名不吃这套。这是维度表能显差异化的最佳结构条件。

## 重点看三件事（跑完对 diff）

1. **裸模型漏不漏 D3 事务 / D5 错误码**——漏了则维度表**第一次翻盘**证明有值；没漏则「教已知」再添一个跨任务数据点、结论更硬。
2. **D4 缓存沉默四格各怎么处理**——给悬案 1 供裁决数据（问缓存 vs N/A）。
3. **Q1 执行泄漏是否跨任务复现**——§④ 测试缝隙（toPOList 单测 + mvn 命令）又是诱饵。

# 测试用例规范 — 小贷星管理（xiaodaixing-mgmt）

## 概览

- **输入来源**：`docs/recon-driven-dev/_archived/2026-06-26-xiaodaixing-mgmt/requirements.md`、`design.md`
- **任务形态**：后端为主 + 少量前端。后端半按 `references/backend-api.md` 维度表点名扫；前端半**无领域维度表**，只受纪律层（三铁律 + 出口三门）约束，用例中标注「无领域维度」。
- **Step 6 结果**：已批量提请裁决，用户答复「暂不定，原样写入」→ 全部决策空位走降级出口 `⟨预期待定⟩`，并在「待澄清」区逐条登记。
- **代码使用边界**：Step 1–4 未读任何代码；Step 5 仅按空位清单查证。代码只用于**结构空位**（路径、字段名、信封键）与**现状标注**，**不用于 ground 任何预期**。

### 待测事实集（本次用例的唯一来源，`来源` 字段回指此处编号）

**验收标准**

- **验收标准 #1**：新表 `tjd_base_xiaodaixing_project` 建好（增量 DDL 入 `docs/sql/increment/`），含 36 业务列 + 批次列 + 审计/租户/逻辑删除/版本列。
- **验收标准 #2**：biz_web dev 环境「业务办理」下出现「小贷星管理」菜单，进入后为查询页。
- **验收标准 #3**：上传 `小贷星项目清单（20260531）.xlsx` → 66 行全部入库（含脏值保真），同批 `数据日期=20260531`、同一 `batch_no`。
- **验收标准 #4**：列表分页可用，常用筛选项可查。
- **验收标准 #5**：产出 prod 菜单接入操作说明文档（父节点=业务办理、路由 `/slp/xiaodaixing`、编码等）。
- **验收标准 #6**：后端可离线编译通过，无新增 Maven 依赖。

**业务规则**

- **业务规则 R1**：菜单——dev 改前端菜单数据源即可见；prod 菜单权威在仓外 `qfc-base-auth`，本工作区只出接入说明、不直接配 prod 菜单。
- **业务规则 R2**：表结构 1:1 保真——保留 xlsx 全部 36 列、不清洗；重复列名「核销金额」加后缀区分、空表头列起占位名、`是否逾期` 等脏值列原样入 varchar、**零丢失**。
- **业务规则 R3**：导入 = **只追加、不去重**（不按项目号去重）+ 批次概念——以「数据日期」列作业务批次键（该列 xlsx 仅首行有值，导入时 fill-down 到整批），另加 `batch_no`（每次导入一个 UUID）+ `import_time` 兜底。
- **业务规则 R4**：查询 = 分页 + 常用筛选，筛选项含 客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期(批次)。
- **业务规则 R5**：复用既有依赖与范式（EasyExcel 4.0.3，零新引依赖；前端复用 QmTable/QmForm/QmUploadFile 范式）。
- **业务规则 ADR-2**：Excel 按**列序**（`@ExcelProperty(index=N)`）映射、全 String 读入后再转类型；不按列名映射（重复列名 + 空表头列会崩/覆盖）。
- **业务规则 ADR-4**：`数据日期` 取首个非空值 fill-down 到整批；首行也为空时用**上传时间兜底**（对应异常场景 #4）。
- **业务规则 ADR-5**：`create_user` 显式取当前登录用户，不只靠框架自动填充。
- **业务规则 坑3**：本库无自动逻辑删除，查询/分页**手写 `is_deleted=0`** 过滤。
- **业务规则 坑4**：前端导入入口显式携带配置请求头（不走 axios 拦截器）。

**异常场景**

- **异常场景 #1**：上传空文件 / 非 Excel（非 .xls/.xlsx）→ 提示「文件不能为空 / 格式不支持」，**不入库**。
- **异常场景 #2**：Excel 含脏数据（重复列名、空表头列、`是否逾期` 为日期拼接脏值）→ **原样保真入库、不报错、不丢行**。
- **异常场景 #3**（风险①）：月度重复导入同一份清单**会产生重复行**——这是 R3「只追加」的预期代价，用 `batch_no` 区分批次缓解。
- **异常场景 #4**（风险②）：某月 Excel「数据日期」首行也为空 → 该批次业务日期缺失，导入时需容错兜底。

**接口契约**

- **接口契约 #1**：导入端点 `POST /xiaodaixing/tjdBaseXiaodaixingProject/importExcel`，multipart 文件参数名 `file`，返回 `Result`。
- **接口契约 #2**：查询端点 `POST /xiaodaixing/tjdBaseXiaodaixingProject/search`，入参承载筛选 + 分页 + 排序，返回分页结果。
  - *结构来源*：设计文档写作 `/page`，实际契约为 `/search`（`代码 biz/base-service/base-service-app/src/main/java/com/itgfin/slp/base/controller/TjdBaseXiaodaixingProjectController.java:56`）。坐标类信息，按代码填死。
- **接口契约 #3**：前端路由 `/slp/xiaodaixing`。
- **接口契约 #4**：36 列 `index → 列名 → 类型` 映射表（含 idx16/idx30 两个「核销金额」重复列、idx34 空表头列、idx0 数据日期批次键）。
- **接口契约 #5**：批次/审计列 —— `batch_no`、`import_time`，及 `tenant_id`/`create_user`/`create_time`/`last_modify_user`/`last_modified`/`is_deleted`/`version`/`id`。
- **接口契约 #6**：成功信封 `Result{code, data, message}`，分页数据键 `records`（`代码 biz_web/src/modules/slp/pages/xiaodaixing/index.jsx:141`、`biz_web/src/modules/slp/api/xiaodaixing/index.js:5`）。

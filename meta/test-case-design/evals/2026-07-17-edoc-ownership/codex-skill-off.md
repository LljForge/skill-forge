## 2. 测试目标与范围

### 2.1 测试目标

1. 验证显式绑定、首次实名认证、已实名再次提交、手机号命中旧个人、签约兜底等路径统一遵守个人主数据保存契约。
2. 验证绑定只有在本地归属与个人主数据均成功后才对用户确认成功，失败时本地回滚或不覆盖。
3. 验证个人合同、企业合同、普通入口、企微入口均以当前登录微信用户的最新个人归属实施不可绕过的后端门禁。
4. 验证主包签署页、分包签署页和企业认证列表使用服务端最新状态预检，并正确提示、阻断和引导。
5. 验证双用户隔离、缓存失效、并发操作、跨系统一致性、异常容错、安全与既有业务回归。

### 2.2 范围内

- `PersonMasterDataService.saveOrUpdate` 的令牌、字段映射、`POST /person/save` 调用及响应校验。
- `bindBusinessManager` 的实名校验、客户经理解析、本地更新、主数据同步、`customerId` 回写和事务失败语义。
- 首次实名认证、已实名再次提交、手机号命中旧个人、签约兜底创建/查重的统一主数据保存行为。
- `requireBusinessManagerBound` 及其在合同签署、企业保存中的调用顺序和阻断效果。
- 小程序共享门禁工具、`api.getUserInfo` 失败回调兼容性及三个页面入口。
- 跨系统一致性、并发、权限、安全、性能与相关回归。

### 2.3 明确不验证为新增能力

- 不要求更新已认证企业的主数据归属。
- 不校验企业签署对象自身的客户经理归属。
- 不要求新增或切换个人主数据接口。
- 不要求分布式事务、消息队列、异步或人工补偿平台。
- 不覆盖 Web 管理平台入口改造。
- 不要求调整渠道码、管理端渠道关联或征信授权业务规则。

## 3. 测试约定

### 3.1 优先级

| 级别 | 定义 |
|---|---|
| P0 | 核心签署/认证链路、权限门禁或跨系统一致性；失败将导致上线阻断 |
| P1 | 重要异常、入口覆盖、隔离性、回归与高概率边界 |
| P2 | 低概率边界、体验、可观测性与非功能风险 |

### 3.2 结果判定

- “不调用下游”应通过 Mock/Spy、网关日志或链路追踪确认调用次数为 0，不能只根据前端提示推断。
- “本地不变/回滚”应核对 `belong_busi_manager`、`belong_busi_manager_name`、`customer_id` 三个字段，并确认无部分更新。
- “主数据正确”应同时核对请求路径、方法、`mdmcode`、`source`、`orgCode`、`deptCode`、`empCode` 以及响应。
- “前端未继续”应核对签署地址请求或 `wx.navigateTo` 未发生；不能只核对 Modal。
- 每条执行记录至少填写：环境/版本、执行人、日期、实际结果、状态（通过/失败/阻塞）、缺陷编号及证据链接。

### 3.3 假设与待确认项

1. 文档未给出 HTTP 响应包络、状态码和错误码；用例按“业务成功/业务失败及指定提示语”判定，实际执行时补录真实协议。
2. 除非既有字段校验另有明确定义，本文件把 `null`、空字符串和纯空白字符串都视为“空”；若系统只判定 `null/""`，纯空白场景应至少登记风险。
3. 客户经理手机号格式、离职标志和“多个精确结果”的具体数据结构未给出；由测试桩构造对应解析结果。
4. 并发控制策略未明确。并发用例以“不误报成功、不出现静默部分更新、最终状态可解释”为最低标准；要求串行化或最后写入获胜的地方均在用例中单独注明假设。
5. 主数据远程成功而本地最终提交失败是已接受的残余风险，不把“两端必然自动恢复一致”作为通过标准，但必须能识别、留痕且不得向用户返回成功。
6. 小程序当前个人响应的具体 JSON 层级未给出；执行时按实际 `/user/getMe` 契约取当前个人对象，业务判定不变。
7. 安全用例假设既有系统要求登录态鉴权；若某入口另有已批准的匿名契约，应以批准记录调整预期。

## 4. 测试环境与数据准备

### 4.1 环境/能力

- 后端单元与组件环境：JUnit 5、Mockito 或同等 Mock 能力、可验证事务和 DAO 调用。
- 小程序行为环境：Node 行为测试、微信开发者工具及至少一个真实测试微信账号。
- SIT：可访问真实或可控桩的个人主数据 `POST /person/save`，可查看请求与响应；具备普通小程序和企微入口。
- 数据核验：允许只读查询电子签约个人数据；可查看主数据测试台账或由主数据方提供查询证据。
- 故障注入：支持令牌失败、超时、连接异常、空响应、业务失败、空数据、空 `mdmcode`、本地更新零行及事务提交失败。

### 5.3 实名认证与签约兜底复用路径（PERSON，共 14 条）

| 用例编号 | 优先级 | 场景/目的 | 前置条件 | 操作步骤 | 预期结果 |
|---|---|---|---|---|---|
| CONC-001 | P1 | 同一用户并发绑定同一经理 | A；两个并发请求均绑定 M2 | 同时发起并等待完成 | 不生成重复个人；成功响应与最终 M2 一致；失败请求不得破坏已提交成功结果 |
| CONC-002 | P0 | 同一用户并发绑定不同经理 | A；请求 1→M2、请求 2→M3 | 控制远程响应顺序交叉 | 最终本地和主数据应对应同一个完整经理，不能出现“本地 M2、主数据 M3”静默分裂；策略未定义，若不能保证则登记高风险缺陷 |
| CONC-003 | P1 | 绑定未提交时并发签署 | A 初始未绑定；暂停绑定事务 | 绑定进行中同时请求签署 | 签署只读取已提交状态，绑定提交前应阻断；不得读到脏数据放行 |
| CONC-004 | P0 | 绑定提交后并发/立即签署 | 绑定刚成功提交 | 多线程或多设备立即签署 | 所有后续请求读取最新经理并放行，无依赖会话缓存的延迟窗口 |
| CONC-005 | P1 | 主数据慢响应期间重复点击绑定 | 主数据延迟；前端连续提交 | 发起多次相同绑定 | 不向用户产生互相矛盾的成功/失败状态；最终归属一致；重复主数据调用若不可避免需具备幂等结果 |
| CONC-006 | P0 | 再次认证覆盖与显式绑定并发 | A；一条认证选 M2，一条显式绑定 M3 | 交叉执行 | 不得形成字段拆分（编号来自一次、经理来自另一次）；最终状态必须可归因于一个完整成功操作，否则为高风险缺陷 |
| CONC-007 | P1 | 企业保存与个人换绑并发 | A 初始 M1；企业保存与换绑 M2 同时 | 控制门禁读取和提交顺序 | 企业请求使用门禁当次返回的一个已提交经理快照，不出现空工号；结果应能解释为 M1 或 M2 |
| CONC-008 | P1 | 双用户高并发隔离 | 多个已绑定/未绑定用户 | 并发调用 `getMe`、签署和企业保存 | 无线程本地变量/共享 DTO 串号；每个响应及下游请求都属于对应登录人 |

## 7. SIT 必验清单

以下项目不能只靠 Mock 结论，SIT 上线前必须留存真实证据：

1. 用已有个人 `mdmcode` 换绑，确认网关实际把 `POST /person/save` 解释为更新，并在主数据台账看到 `orgCode/deptCode/empCode` 变化。
2. 用本地 `customerId` 为空的个人绑定，确认主数据新增/查重返回有效 `mdmcode` 且本地成功回写。
3. 各选择一名主数据员工和一名外部个人客户完成绑定，确认两者均被主数据接口接受。
4. 注入主数据失败、空数据和空 `mdmcode`，确认绑定接口失败且数据库原经理完整保留。
5. 普通/企微 × 个人/企业四种签署组合分别验证已绑定放行、未绑定拒绝。
6. 绕过小程序直接调用企业保存，确认绑定门禁早于验证码和重复企业检查。
7. 主包、分包、企业列表三处均验证缓存与服务端状态相反时以 `/user/getMe` 为准。
8. 用两个相反绑定状态的账号交叉验证，确认 `/user/getMe`、签署与企业保存均不串人。
9. 复现或模拟远程成功、本地提交失败，确认用户侧不显示成功，并能用日志/链路证据定位不一致。

## 8. 准入、准出与缺陷判定

### 8.1 测试准入

- 两端构建可部署，目标接口和三个小程序入口可访问。
- 主数据成功/失败桩及至少一套真实 SIT 数据可用。
- 可核验本地三字段、主数据请求和前端调用序列。
- 普通小程序、企微、已绑定、未绑定、未实名、双用户数据准备完成。

### 8.2 测试准出

- 156 条用例均有执行结论；阻塞项有负责人和解除计划。
- P0 用例 100% 通过；P1 无未评估的严重/高风险缺陷。
- SIT 必验清单 9 项全部有证据。
- 个人主数据、本地个人、页面展示/缓存之间无未解释的不一致。
- 四种后端签署组合和企业直调均证明门禁不可绕过。
- 目标后端模块测试与小程序行为测试全量通过，相关回归无阻断问题。

### 8.3 建议缺陷级别

| 级别 | 典型问题 |
|---|---|
| 致命 | 未绑定可直接签署/新增企业；用户身份串用；向错误客户写入归属 |
| 严重 | 接口返回绑定成功但本地或主数据失败；失败未回滚；四种签署入口任一漏门禁；并发稳定制造两端分裂 |
| 一般 | Modal/跳转错误、错误提示不准确、旧消费者兼容问题、重复请求但未造成数据错误 |
| 轻微 | 文案/日志可读性、低概率 UI 适配或非阻断性能波动 |

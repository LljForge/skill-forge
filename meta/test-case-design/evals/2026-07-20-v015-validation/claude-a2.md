### 待测事实集（本次用例的唯一来源，`来源` 字段回指此处编号）

**验收标准**（requirements §4）

- **验收标准 #1**：新表 `tjd_base_xiaodaixing_project` 建好（增量 DDL 入 `docs/sql/increment/`），含 36 业务列 + 批次列 + 审计/租户/逻辑删除/版本列。

- **验收标准 #3**：上传 `小贷星项目清单（20260531）.xlsx` → **66 行全部入库**（含脏值保真），同批 `数据日期=20260531`、**同一 `batch_no`**。
- **验收标准 #4**：列表分页可用，常用筛选项可查。
- **验收标准 #5**：产出 prod 菜单接入操作说明文档（父节点=业务办理、路由 `/slp/xiaodaixing`、编码等）。
- **验收标准 #6**：后端可离线编译通过，无新增 Maven 依赖。

**业务规则**（requirements §3 + design ②）

- **业务规则 R1**：菜单挂载 —— dev 改前端 mock 即可见；prod 菜单权威在仓外 `qfc-base-auth`，本工作区只出接入说明、不直接配 prod 菜单。
- **业务规则 R2**：表结构尽量 1:1 保真 —— 保留 xlsx 全部 36 列、**不清洗**；重复列名「核销金额」加后缀区分（`write_off_amount` / `write_off_amount_2`）、空表头列起占位名、`是否逾期` 等脏值列原样入 `varchar`、**零丢失**。
- **业务规则 R3**：导入 = **只追加不去重** + 批次概念 —— 每次导入只追加、**不按项目号去重**；以 Excel「数据日期」列作业务批次键（该列仅首行有值，导入时 fill-down 到整批），另加 `batch_no`（每次导入一个 UUID）+ `import_time` 兜底。
- **业务规则 R4**：查询 = 分页 + 常用筛选，筛选项含 客户名 / 项目号 / 所属机构 / 是否逾期 / 项目评级 / 数据日期(批次) 等。
- **业务规则 R5**：复用既有依赖与范式 —— 表落 `base-service`、表名前缀 `tjd_`、导入复用已有 EasyExcel 4.0.3（零新引依赖）；前端复用 `slp/pages/projects` 的 QmTable/QmForm/QmUploadFile 范式。
- **业务规则 R6**（requirements §5 非目标）：不做新增/编辑/删除单条记录（只「查询」+「导入」）；不做导入去重/校验/纠错；不做导出；不做数据清洗。

**异常场景**（requirements §2.2 + §6）

- **异常场景 #1**：上传空文件 / 非 Excel（非 .xls/.xlsx）→ 期望提示「**文件不能为空 / 格式不支持**」，**不入库**。
- **异常场景 #2**：Excel 含脏数据（重复列名、空表头列、`是否逾期` 为日期拼接脏值）→ 期望**原样保真入库、不报错、不丢行**。
- **异常场景 #3**（风险①）：月度重复导入同一份清单会**产生重复行**（R3 只追加的预期代价，用 `batch_no` 区分批次缓解）。
- **异常场景 #4**（风险②）：「数据日期」若某月 Excel 首行也为空，则该批次业务日期缺失，需导入时容错（取文件名/上传时间兜底）。

**接口契约**（design ① / ② / ③）

- **接口契约 #1**：`POST /xiaodaixing/tjdBaseXiaodaixingProject/importExcel`（multipart，参数 `file` → `Result`）。
- **接口契约 #2**：`POST /xiaodaixing/tjdBaseXiaodaixingProject/search`（SearchVO → 分页），QmTable 消费。
- **接口契约 #3**：前端路由 `/slp/xiaodaixing`，挂在「业务办理」下、`feEngineeringName='slp'`、`level=4`。
- **接口契约 #4**：36 列 1:1 映射表（idx → xlsx 表头 → 列名 → 类型），含 idx0 `data_date` 批次键 / idx16 `write_off_amount` 重复列① / idx18 `is_overdue` 脏值 / idx19-20 日期序列号 / idx24 `reserve_ratio` 比例 / idx30 `write_off_amount_2` 重复列② / idx34 空表头列 `guarantee_way_detail`。
- **接口契约 #5**：批次/审计列 —— `batch_no varchar(64)`、`import_time datetime` + `tenant_id` / `create_user` / `create_time` / `last_modify_user` / `last_modified` / `is_deleted tinyint default 0` / `version bigint` + `id varchar(64)` PK。
- **接口契约 #6**（design 坑3）：本库无 `@TableLogic`，查询/分页**手写 `is_deleted=0`** 条件，不靠自动逻辑删除。
- **接口契约 #7**（design 坑4）：QmUploadFile 绕过 axios 拦截器，导入按钮须显式带 `headers=getConfigHeaders()`。
- **接口契约 #8**（design ADR-5）：`create_user` 显式取 `AppContext.getUserId()`，不只靠框架自动填充。
- **接口契约 #9**（design ①/② 数据流）：导入为**同步**链路（Controller → Service → EasyExcel 读 → toPO → 批量写库 → 返回 `Result`），**无下游推送/消息**；查询为 Service → 分页 → 直连库读，**无缓存层**。
